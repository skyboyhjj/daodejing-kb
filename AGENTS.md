功能实现完成后，必须逐条对照设计文档检查：

- [ ] 数据模型字段名称和类型是否与设计文档一致
- [ ] 阈值、触发条件等配置参数是否完全匹配
- [ ] 命名约定（阶段名、变量名、函数名）是否与设计统一
- [ ] 所有标注为"必须"的设计约束是否已满足
- [ ] 是否遵循了宪法第七、十一、十四条
- [ ] 是否修改了不必要的后端代码（外挂增强原则）

## 三、已知偏差案例

SkillUP 认知成长追踪实现偏离设计文档的 12 处偏差（反面案例）：

1. **成长模型隐喻**：设计为"水德"（水滴→溪流→江河→江海），实现为"植物"（萌芽→小苗→树木→森林）
2. **阶段阈值**：设计 10/30/60，实现 3/10/30
3. **触发规则缺失**：设计 4 条（阅8章后/L3切换/连续3天/主动查询），实现仅 2 条（数字里程碑+主动查询）
4. **连续天数**：设计为真正连续检测，实现为首次至今总天数差
5. **独立许可文件**：设计要求 `skillup-consent.js`，实现内嵌于 `skillup-ui.js`
6. **数据模型 keyword 字段**：设计含 keyword 提取，实现无
7. **数据模型 chapter 关联**：设计 questions 关联章节，实现 messages 无 chapter
8. **数据模型 chaptersVisited**：设计为数组，实现为对象
9. **章节页面底部引导**：设计要求，未实现
10. **CSS 视觉效果**：实现了设计系统禁用的 backdrop-filter blur + box-shadow
11. **lastVisit 字段**：设计有 lastVisit，实现用 firstVisit
12. **totalChaptersRead**：设计独立字段，实现动态计算

## 四、项目技术约定

- **外挂增强**：新增功能应优先使用外部模块（新 JS/CSS 文件），最小化对现有核心代码的修改
- **零服务端原则**：用户数据使用 localStorage，不上传服务器（宪法第十一条）
- **水墨道风设计**：配色 #2e5f5f（墨绿）/ #d4c9a8（淡金）/ #faf8f0（宣纸），无 box-shadow、无 blur、无 gradient
- **文件命名**：新文件使用小写+连字符（kebab-case）
- **JS 模块**：使用 IIFE 模式 `(function() { 'use strict'; ... })();`
- **Node.js**：脚本使用 Node.js ≥16 标准库
