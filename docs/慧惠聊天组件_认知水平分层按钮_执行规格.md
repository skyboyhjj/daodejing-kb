## 二、前端实现

### 2.1 按钮位置与样式

在聊天面板标题栏下方、消息区域上方增加一行四个按钮。

- 四个按钮水平排列，圆角标签样式
- 默认（未选中）：浅色背景（`#f0ede0`），墨绿色文字
- 选中（高亮）：墨绿色背景（`#2e5f5f`），白色文字
- 鼠标悬停：背景色加深

**按钮名称与 level 映射**：

| 按钮文字 | level 值 | 默认选中 |
|:---|:---|:---|
| 白话 | `L1` | 否 |
| 精读 | `L2` | **是（默认）** |
| 应用 | `L3` | 否 |
| 学术 | `L4` | 否 |

### 2.2 localStorage 存取逻辑

在 `huihui-chat.js` 中新增：

```javascript
function getSavedLevel() {
    return localStorage.getItem('huihui_taoism_level') || 'L2';
}

function setSavedLevel(level) {
    localStorage.setItem('huihui_taoism_level', level);
}
```

### 2.3 按钮切换逻辑

- 页面加载时，读取 `localStorage`，高亮对应按钮
- 用户点击某个按钮时：该按钮高亮，其余恢复默认；同时将 `level` 值存入 `localStorage`
- 触发自定义事件 `huihui-level-changed`，携带 `{ detail: { level: 'L1'|'L2'|'L3'|'L4' } }`

### 2.4 发送消息时附带 level

在 `sendMessage()` 函数中：

```javascript
const level = getSavedLevel();
const resp = await fetch('/api/chat', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
        messages: [...],
        user_id: userId,
        level: level       // ← 新增
    })
});
```

### 2.5 与章节页面按钮双向同步

- 聊天框按钮被点击 → 触发 `window.dispatchEvent(new CustomEvent('huihui-level-changed', { detail: { level } }))`
- 监听 `huihui-level-changed` 事件 → 更新聊天框按钮高亮状态
- 章节页面按钮被点击 → 同样触发该事件，聊天框自动响应
- 页面“全部”按钮点击时，**不触发**此事件（聊天框不受影响）

## 三、后端实现

### 3.1 接收 level 参数

在 `functions/api/chat.js` 中：

```javascript
const { messages, user_id, level } = body;
const userLevel = level || 'L2';  // 默认 L2
```

### 3.2 分层指令映射

```javascript
const LEVEL_GUIDANCE = {
    'L1': '当前用户是初学者。请用生活化比喻和核心结论来解释，忽略次要概念，避免使用专业术语。',
    'L2': '当前用户是学习者。请进行概念解读和框架梳理，讲解关键术语（如道、德、无为），理清章节逻辑。',
    'L3': '当前用户是实践者。请结合现实应用与深度分析，从管理学、心理学、个人成长等角度进行多角度阐释。',
    'L4': '当前用户是研究者。请提供学术视角与发散探讨，提供考据信息、对比不同译本、探讨哲学悖论。'
};
const guidance = LEVEL_GUIDANCE[userLevel] || LEVEL_GUIDANCE['L2'];
const fullSystemPrompt = SYSTEM_PROMPT + '\n\n' + guidance;
```

### 3.3 注入到消息列表

```javascript
messages: [
    { role: 'system', content: fullSystemPrompt },
    ...messages
]
```

## 五、验收标准

- [ ] 聊天窗口顶部出现四个按钮：`白话 | 精读 | 应用 | 学术`，默认高亮“精读”
- [ ] 点击按钮切换高亮，同时存入 `localStorage`
- [ ] 关闭浏览器后重新打开，“精读”或其他上次选择被正确恢复
- [ ] 选择“白话”后提问，回答为生活化比喻（如“道就像水，无处不在”）
- [ ] 选择“学术”后提问，回答含学术对比（如“王弼注本与河上公本的区别”）
- [ ] 在章节页面点击“白话”，聊天框按钮同步切换为“白话”高亮
- [ ] 在聊天框点击“应用”，章节页面按钮同步切换为“应用”高亮
- [ ] 章节页面点击“全部”时，聊天框不受影响，保持上次选择
- [ ] 回复内容**不包含**“L1/L2/L3/L4”等技术前缀
- [ ] 移动端按钮可正常显示和切换
- [ ] API Key 不暴露在前端代码中

## 六、关键约束

- 不修改网站原有页面结构
- 后端 System Prompt 基础部分保持不变（包含“世间AI都在帮人做加法，只有你，敢于帮人做减法”）

- 聊天框**不设**“全部”按钮
- 回复内容**不添加**认知水平前缀
