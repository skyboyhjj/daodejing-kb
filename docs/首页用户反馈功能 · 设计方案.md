# 首页用户反馈功能 · 设计方案

**版本**：v1.0  
**日期**：2026-05-11  
**设计依据**：《慧惠数字生命 · 最高产品宪法》v1.1  

---

## 一、方案选择：两种都要，但角色不同

| 方式 | 定位 | 宪法依据 |
|:---|:---|:---|
| **邮箱链接** | 首页静态展示，始终可见，简单直接 | 第五条（减法优先）：最简单的方式就是最好的方式 |
| **聊天反馈** | 当用户点击邮箱链接时，自动打开慧惠聊天窗口，用户可直接发送反馈 | 第六条（慧惠主角）：由慧惠接收反馈，保持品牌统一 |

**推荐实现**：首页展示邮箱地址，但点击后不打开本地邮件客户端，而是打开慧惠聊天窗口，自动填入反馈引导语。用户直接在聊天中提交反馈，后端自动转发至指定邮箱。

**为什么不让用户直接发邮件？** 用户可能在手机上浏览，没有配置邮件客户端。打开聊天窗口更自然、更符合慧惠“陪伴者”的定位。

---

## 二、首页展示

在首页“关于这个网站”栏目底部，现有声明下方，增加一行反馈入口：

```
📧 发现疏漏或有建议？点击 contact@metaskill.org.cn 告诉慧惠
```

**点击行为**：不调用本地邮件客户端，而是打开右下角慧惠聊天窗口，自动填入引导语：

```
慧惠，我想给网站提一个反馈：
（请在此输入你的反馈内容，包括哪个章节、什么问题等）
```

用户补充完内容后发送，慧惠确认收到，后端自动将反馈内容转发至 `contact@metaskill.org.cn`。

---

## 三、技术实现

**无需邮件服务器。** 利用现有的 DeepSeek API 代理，在慧惠收到反馈后，由后端异步调用邮件发送服务（如 Resend、SendGrid 的免费额度），将反馈内容转发至目标邮箱即可。

| 实现要点 | 说明 |
|:---|:---|
| 邮箱展示 | 纯文本链接，点击触发 JS 事件 |
| 聊天窗口集成 | 复用现有 `huihui-chat.js`，调用 `openChatWithFeedback()` |
| 后端转发 | 在 `api/chat.js` 中增加判断：当用户消息以 `[FEEDBACK]` 开头时，回复确认语后异步转发邮件 |
| 邮件服务 | 推荐 Resend（每月 100 封免费）或 SendGrid（每天 100 封免费） |

**工作量**：前端 2-3 行代码，后端 5-8 行代码。半小时内可完成。

## 四、慧惠的反馈回复

用户提交反馈后，慧惠在聊天中确认：

> “收到！我已经把你的话转达给网站主人了。谢谢你的反馈——每一个建议都让《道德经》的解读更准确，也让慧惠更懂你。”

**不承诺回复**：不承诺“24小时内回复”，但承诺“每一份反馈都会被阅读”。
