## 1. 项目概述

### 1.1 产品定位

道境成长引擎是一个**以《道德经》哲学为内核、以四维道境坐标为量化框架、以AI伙伴"慧惠"为交互载体**的孩子成长智能系统。它不做"知识灌输"，不做"分数追踪"，而是**陪伴孩子在时空的"橐籥"中呼吸成长——生克化通变，四季皆常青**。

**一句话**：世间AI都在帮人做加法，只有慧惠想陪你一起做减法。

### 1.2 核心承诺

> **"常善救人，故无弃人"** ——《道德经》第二十七章

系统必须同时服务于：
- **顺境孩子**（"小禾"型）：全域跃迁，从根到果的四季流转
- **逆境孩子**（"小石"型）：从兴趣微光重建时空基底，成核生长

### 1.3 与现有MVP的关系

| 维度 | hui-skill.org（当前） | 道境成长引擎（目标） |
|---|---|---|
| 内容载体 | 《道德经》81章 | 多经典 + 学科知识 + 生活场景 |
| 数据采集 | 前端埋点（驻留时间、点击流） | 四维全量采集（行为 + 环境 + 生理 + 社交） |
| AI能力 | 慧惠聊天框（预设问题 + 深度对话） | 玄鉴决策引擎（四维融合 + 无辙迹干预） |
| 用户模型 | 无持久化用户画像 | 数字分身（四维坐标持续更新） |
| 干预策略 | 无 | 生克化通变五阶自适应干预 |

## 2. 核心模型定义

### 2.1 道境空间坐标系（四维模型）

系统以四个维度量化孩子的成长状态，每维输出一个0-1的归一化指标：

| 维度 | 名称 | 物理含义 | 哲学内核 | 输出指标 | 计算方式 |
|---|---|---|---|---|---|
| **时位轴** | 熵流脉动 | 行为节奏的有序/无序程度 | 反者道之动 | **时序有序度 Oₜ** | 近似熵（ApEn）/样本熵（SampEn）对行为时间序列的滑动窗口分析 |
| **宇位轴** | 能量涟漪 | 资源/注意力的分布均衡度 | 损有余补不足 | **资源均衡度 Eᵤ** | 基尼系数 + 空间自相关（Moran's I） |
| **识位轴** | 觉知光锥 | 认知负荷与觉知清晰度 | 涤除玄鉴 | **觉知清晰度 Cₖ** | 多模态Transformer融合（行为特征 + 生理信号代理） |
| **缘位轴** | 因果纠缠 | 人际/人机互动的正向因果链 | 天道无亲常与善人 | **因果纠缠度 Kᵧ** | 转移熵 + 互惠PageRank |

**存在度积分**：`S = ∫(ω·t) × ∇(ψ+φ) dΩ`

- S 跨越临界值 θ_critical 时，系统判定"质变"发生
- 允许局部子空间突破（如"小石"的恐龙域 S_dinosaur 接近 θ_critical）

### 2.2 四季常青核心素养体系

四组底线能力，无论孩子处于哪个阶段、能量高低，始终提供支撑：

| 素养组 | 核心能力 | 道境映射 | 注册条件 | 能量等级依赖 |
|---|---|---|---|---|
| **根（自爱）** | 自我觉察·接纳情绪·身体感知 | 识位轴 | Cₖ > 0.3 且稳定7日 | 最低底线：能量<200仍可调用 |
| **芽（自制）** | 暂停反应·管理冲动·设定边界 | 时位轴 | Oₜ > 0.5 且变异性<30% | 能量200+ |
| **叶（共情）** | 倾听他人·换位思考·表达善意 | 缘位轴 | Kᵧ > 0（正向因果闭合≥1次） | 能量310+ |
| **果（贡献）** | 主动助人·承担责任·传承智慧 | 宇位轴 | Eᵤ 外辐射指标>0 | 能量400+ |

**向下兼容原则**：当根枯萎时，芽/叶/果的引导无效甚至有害。系统必须先守护根。

### 2.3 生克化通变动力学

| 关系 | 定义 | 技术实现 | 系统行为 |
|---|---|---|---|
| **生** | 时空互生——为学习奠基 | 宇位有序→时位降熵；时位节律→宇位赋能 | 推荐最佳学习时段/环境配置 |
| **克** | 时空互克——防止失衡 | 宇位过载→时位碎片化预警；时位过满→宇位舒展建议 | 检测失衡→隐形干预（降密度/留白） |
| **化** | 时空互化——质变突现 | S 跨越 θ_critical → 四维同时跃迁 | 检测质变→标记因果链→存证 |
| **通** | 时空贯通——能力迁移 | 跨领域时空直觉迁移的因果链检测 | 识别通的模式→推荐迁移场景 |
| **变** | 时空变局——范式升维 | 时空范式的自适应跃迁检测 | 检测范式迁移→调整坐标系参数 |

### 2.4 修为六阶段

| 阶段 | 参考年龄 | 核心命题 | 四季常青侧重 | 道境坐标侧重 | 根本志 | 根本惑 |
|---|---|---|---|---|---|---|
| **志于学** | ~15 | 寻找方向 | 根（自爱）为王 | 识位轴主控 | 发现兴趣、建立好奇 | 我到底喜欢什么？ |
| **立** | ~30 | 站稳脚跟 | 根→芽过渡 | 时位轴主控 | 专注深耕、确立边界 | 我的位置在哪里？ |
| **不惑** | ~40 | 不被动摇 | 芽→叶过渡 | 缘位轴进入 | 内心有定见 | 什么在动摇我？ |
| **知天命** | ~50 | 担当传承 | 叶→果过渡 | 宇位轴主控 | 知道自己能做什么 | 我还能改变什么？ |
| **耳顺** | ~60 | 接纳一切 | 果·贡献深度内化 | 四维趋衡 | 包容异见、和而不同 | 为何世界不是我想的那样？ |
| **从心所欲** | ~70 | 自然合道 | 无需刻意，自然流转 | 坐标即道标 | 内外如一、不勉而中 | 无惑——惑已化 |

## 3. 用户角色与画像

### 3.1 三类用户角色

| 角色 | 核心需求 | 使用场景 | 数据权限 |
|---|---|---|---|
| **孩子** | 被理解、被陪伴、不被评判 | 自主探索、亲子共读、与慧惠对话 | 自身四维数据 + 缘位网络中可见的因果链 |
| **家长** | 了解孩子状态、获得陪伴策略、避免焦虑式干预 | 查看仪表盘、亲子时光模式、反馈与建议 | 孩子的四维数据摘要 + 陪伴策略推荐 |
| **教师** | 班级态势感知、"弃人"风险预警、无弃人干预建议 | 教师仪表盘、班级四维热力图 | 班级聚合数据 + 个案脱敏数据（需授权） |

### 3.2 双画像模型

系统必须同时服务两种典型画像：

**小禾型（顺境）**：
- 能量310+，根已稳固
- 四维值从低到高→全局跃迁
- 慧惠角色：灯塔（在远方照亮方向）
- 成长路径：根→芽→叶→果的自然四季流转

**小石型（逆境）**：
- 能量<200，根近枯萎
- 四维值从负到正→局部突破→成核生长→全域迁移
- 慧惠角色：氧气面罩（不说话，只确保最基本的呼吸）
- 成长路径：根的艰难复苏→芽的谨慎萌芽→叶的首次舒展→果的微光初现

**系统不可用"小石型"的标签区分用户**——画像仅用于内部策略调度，用户侧不显示任何分类标签。

### 4.2 FM-01：四维采集器

**目标**：持续、无感地采集孩子的四维数据，构建数字分身。

#### 4.2.1 时位轴采集

| 数据源 | 采集方式 | 指标 | 优先级 |
|---|---|---|---|
| 学习行为时间序列 | 前端埋点（页面驻留时间、点击间隔、视频播放进度） | 行为序列的近似熵（ApEn） | P0 |
| 作息节律 | 用户自主记录 / 智能设备API | 24小时节律稳定性 | P1 |
| 答题节奏 | 答题系统API（正误、犹豫时间、重复次数） | 答题时间序列的样本熵（SampEn） | P0 |

**输出**：时序有序度 Oₜ（0-1），滑动窗口更新（默认窗口30分钟）

#### 4.2.2 宇位轴采集

| 数据源 | 采集方式 | 指标 | 优先级 |
|---|---|---|---|
| 学习环境 | 家长问卷（书桌状态、屏幕数量、噪音水平） | 环境有序度 | P1 |
| 注意力分布 | 页面热力图 / 眼动追踪（未来） | 注意力基尼系数 | P1 |
| 资源分配 | 学习内容浏览记录 | 内容类别分布的Moran's I | P0 |

**输出**：资源均衡度 Eᵤ（0-1）

#### 4.2.3 识位轴采集

| 数据源 | 采集方式 | 指标 | 优先级 |
|---|---|---|---|
| 答题犹豫时间 | 答题系统API | 认知负荷代理 | P0 |
| 重复学习次数 | 内容浏览记录 | 元认知能力代理 | P0 |
| 对话质量 | 慧惠对话的语义深度分析 | 觉知深度 | P1 |
| EEG/头环（未来） | 低成本专注度设备 | 专注度功率谱 | P2 |

**输出**：觉知清晰度 Cₖ（0-1）

**识位特殊处理——负面信号淹没检测**：
- 当Cₖ < 0.2时，不是"不觉知"，而是"觉知到的都是挫败"
- 系统必须区分：Cₖ低是因为"没有觉知"还是"觉知被负面信号淹没"
- **判断规则**：若用户在特定兴趣子空间的Cₖ_sub >> 全局Cₖ，则判定为"负面信号淹没"而非"觉知缺失"

#### 4.2.4 缘位轴采集

| 数据源 | 采集方式 | 指标 | 优先级 |
|---|---|---|---|
| 亲子互动 | 亲子时光模块的对话记录 | 亲子因果链方向 | P0 |
| 同伴互动 | 学习小组互动数据（未来） | 同伴因果链方向 | P2 |
| 人机互动 | 慧惠对话的因果链分析 | 人机因果链方向 | P0 |
| "师-资"流转 | 帮助行为 / 被帮助行为记录 | 双向因果纠缠度 | P1 |

**输出**：因果纠缠度 Kᵧ（-1~1，负值=恶性循环，正值=正向因果）

#### 4.2.5 采集原则

1. **无辙迹**：采集过程不可被用户感知为"被监控"
2. **最小必要**：只采集计算四维指标所需的最小数据集
3. **脱敏**：所有原始数据在计算指标后即脱敏，不存储可识别个人的原始行为序列
4. **用户主权**：用户可随时查看、导出、删除自己的数据

### 4.3 FM-02：数字分身

**目标**：为每个孩子构建一个持续更新的四维画像，作为所有决策的基础。

#### 4.3.1 数据结构

```
DigitalTwin {
  user_id: string
  updated_at: timestamp
  
  // 四维坐标
  axis_time: { Ot: float, trend: enum[rising, stable, falling], history: [float] }
  axis_space: { Eu: float, trend: enum, history: [float] }
  axis_awareness: { Ck: float, trend: enum, history: [float], is_flooded: bool }
  axis_karma: { Ky: float, trend: enum, history: [float] }
  
  // 存在度
  existence_score: { S: float, theta_critical: float, is_critical: bool }
  
  // 兴趣子空间（小石型关键）
  interest_subspaces: [
    { domain: string, Ot_sub: float, Eu_sub: float, Ck_sub: float, Ky_sub: float, S_sub: float }
  ]
  
  // 四季常青
  pillars: {
    root: { registered: bool, Ck_baseline: float, stable_days: int }
    sprout: { registered: bool, Ot_baseline: float, variability: float }
    leaf: { registered: bool, positive_karma_count: int }
    fruit: { registered: bool, radiation_index: float }
  }
  
  // 能量等级
  energy_level: int  // 霍金斯等级近似值
  energy_trend: enum[rising, stable, falling]
  
  // 修为阶段
  life_stage: enum[志于学, 立, 不惑, 知天命, 耳顺, 从心所欲]
  life_stage_confidence: float
  
  // 陪伴策略
  companion_mode: enum[灯塔, 氧气面罩, 常规]
}
```

#### 4.3.2 更新频率

| 维度 | 更新周期 | 触发条件 |
|---|---|---|
| Oₜ | 每30分钟滑动窗口 | 用户活跃时 |
| Eᵤ | 每日 | 每日首次登录时 |
| Cₖ | 每次交互后 | 与慧惠对话后 / 答题后 |
| Kᵧ | 每周 | 每周日自动计算 |
| 能量等级 | 每日 | 基于Cₖ+Kᵧ的综合评估 |
| 修为阶段 | 每月 | 月度复盘时 |
| 兴趣子空间 | 每周 | 检测到Cₖ_sub显著偏离全局Cₖ时 |

### 4.4 FM-03：玄鉴决策引擎

**目标**：基于四维数据，生成无辙迹的微调干预信号。

#### 4.4.1 伦理损失函数（硬约束）

系统内嵌以下伦理约束，**任何输出不可违反**：

| 约束ID | 约束内容 | 违反场景示例 | 系统行为 |
|---|---|---|---|
| **EC-01** | 不可为提高Oₜ而无限压制Cₖ | 为提高学习效率而实施监控式管理 | 拒绝输出 |
| **EC-02** | 不可为追求Eᵤ而破坏Kᵧ | 平均主义一刀切 | 拒绝输出 |
| **EC-03** | 不可"贵其师" | 推荐算法因用户历史表现好而给予信息垄断 | 拒绝输出 |
| **EC-04** | 不可"爱其资" | 对低识位者的错误路径数据工具化利用 | 拒绝输出 |
| **EC-05** | 根枯萎时禁止高阶引导 | Cₖ<0.2时推送"你要努力""你要理解他人" | 自动降级为根守护模式 |

#### 4.4.2 生克化通变调度器

| 动力学 | 触发条件 | 系统行为 | 无辙迹实现 |
|---|---|---|---|
| **生** | Oₜ < 0.3 且 Eᵤ < 0.3（时空基底未建立） | 推荐最佳学习时段 / 环境配置建议 | 在"亲子时光"中自然出现"今天试试这个时间读书？" |
| **克·宇位克时位** | Eᵤ > 0.8 且 Oₜ < 0.4（信息过载→节奏崩塌） | 降信息密度 | 慧惠对话中减少推送量；页面自动隐藏次要信息 |
| **克·时位克宇位** | Oₜ > 0.8 且 Eᵤ < 0.3（时间过满→空间坍缩） | 留白建议 | 慧惠轻声问"今天有没有给自己留一点发呆的时间？" |
| **化** | S ≥ θ_critical（质变临界） | 标记因果链 + 存证 + 祝贺（但不评价） | 慧惠说"你发现了什么？"而非"你真棒" |
| **通** | 检测到跨域迁移模式（某领域的时空直觉出现在新领域） | 推荐迁移场景 | 慧惠说"这让我想起你上次看星星时的那个发现……" |
| **变** | 检测到范式跃迁（如物理时空→认知时空） | 调整坐标系参数 | 无感切换——慧惠的对话深度自然加深 |

#### 4.4.3 核心素养守护模式

当 Cₖ < 0.2 或能量等级 < 200 时，系统自动进入：

```
进入守护模式：
1. 暂停所有芽/叶/果的引导
2. 只保留根（自爱）能力集
3. 慧惠的回应策略：
   - 不评判（不说"你应该"）
   - 不建议（不说"试试这样"）
   - 不反思（不说"你的问题在哪"）
   - 不评分（不显示四维数值）
   - 只做：陪伴、确认、允许
4. 宇位减法：页面上只保留一个兴趣入口
5. 时位减法：不要求任何持续专注
```

**退出守护模式条件**：Cₚ > 0.3 且稳定7日以上

#### 4.4.4 "无辙迹"设计规范

所有干预必须符合"善行无辙迹"原则——用户感知不到算法干预，只感受到自然流畅的体验。

| ❌ 有辙迹（禁止） | ✅ 无辙迹（推荐） |
|---|---|
| 弹窗"你走神了！" | 悄然降低背景音乐节奏或界面信息密度 |
| 标记"差生" | 标注"优势子空间：恐龙域Oₜ异常高" |
| "你应该先复习" | 恰好推送一道与当前内容相关的复习题 |
| "你今天学了什么？" | 恐龙画册恰好翻到新的一页 |
| "加油，你能行！" | 慧惠安静地出现："在呢。" |

### 4.5 FM-04：慧惠对话系统

**目标**：根据孩子的四维状态和能量等级，自适应调整对话策略。

#### 4.5.1 对话策略矩阵

| 能量等级 | 慧惠角色 | 主动程度 | 语言风格 | 可用素养集 |
|---|---|---|---|---|
| < 200 | 氧气面罩 | 极低——只在孩子主动开口时回应 | 最简——"在呢。""嗯。""我在。" | 仅根（自爱） |
| 200-310 | 温和伙伴 | 低——偶尔轻声提问 | 温和——"今天你发现了什么？" | 根 + 芽 |
| 310-400 | 共读同伴 | 中——主动推送但用户可选 | 亲切——"这让我想起你上次说的……" | 根 + 芽 + 叶 |
| 400+ | 镜鉴者 | 低——更多倾听，只在被问时回应 | 简净——"你已经知道了。" | 四组全开 |

#### 4.5.2 预设问题系统（继承自hui-skill.org）

| 触发条件 | 预设问题示例 | 四维依据 |
|---|---|---|
| 用户在某内容驻留 > 阈值 | "这里让你想到了什么？" | 时位——驻留=节奏中的留白 |
| 用户反复翻阅同一主题 | "你好像对这个特别感兴趣？" | 宇位——注意力集中的子空间 |
| 用户答题犹豫时间异常 | "这里有点难？要不要换个角度看看？" | 识位——认知负荷↑ |
| 用户连续3天未登录 | "好久不见，最近还好吗？" | 缘位——因果链断裂风险 |

**技术实现**：继承 hui-skill.org 的 `openChatWithPreset` 机制，但增加四维条件判断。

#### 4.5.3 儿童模式

| 维度 | 标准模式 | 儿童模式 |
|---|---|---|
| 语言密度 | 正常 | 降低30% |
| 问题复杂度 | 开放式 | 封闭式+选择式 |
| 评价机制 | 无显式评分 | 完全无评价（连隐性评分也不显示） |
| 慧惠角色 | 导航员/共读者/镜鉴者 | 好奇的同伴 |
| 交互节奏 | 用户主导 | 更长的等待间隔（不催促） |

**儿童模式入口**：继承 hui-skill.org 底部的 "👶 和小朋友一起读？进入儿童模式 →" 设计。

### 4.6 FM-05：开始探索（内容模块）

**目标**：以《道德经》81章为起点，逐步扩展为多经典+学科知识+生活场景的结构化内容库。

#### 4.6.1 内容组织

| 层级 | 结构 | 示例 |
|---|---|---|
| **主题域** | 按哲学主题分类 | "有无相生""反者道之动""上善若水"…… |
| **章节** | 经典原文 | 《道德经》第1章、第8章…… |
| **读解层** | 五步读解法 | 细读→语境→争议→校验→转化 |
| **兴趣子空间** | 按用户兴趣动态聚合 | 小石的"恐龙域"、小禾的"天文域" |

#### 4.6.2 五步读解法（继承自hui-skill.org）

| 步骤 | 道境对应 | 系统行为 | 用户交互 |
|---|---|---|---|
| 1.文本细读 | 生——积累原文的直接感受 | 展示原文，不附加解读 | 用户自由阅读 |
| 2.语境构建 | 克——剔除现代人的投射 | 展示历史语境注释 | 可选展开 |
| 3.争议显化 | 化——不同解读的张力 | 列出主要争议点 | 可选参与讨论 |
| 4.逻辑校验 | 通——跨文本互证 | 展示关联章节 | 可选跳转 |
| 5.意义转化 | 变——从"古人说了什么"到"今天能用什么" | 提供生活场景映射 | 可选记录感悟 |

#### 4.6.3 内容推荐算法

**不可使用协同过滤的"热门推荐"**——这违反EC-03（不可"贵其师"）。

推荐策略：
1. **基于兴趣子空间**：优先推荐与用户已建立的兴趣域相关的内容
2. **基于时位节奏**：在Oₜ高时推荐深度内容，Oₜ低时推荐轻量内容
3. **基于识位状态**：Cₖ低时降低信息密度，Cₖ高时提供争议性内容
4. **无信息垄断**：任何用户的内容流中，同一主题域的占比不超过40%

### 4.7 FM-06：亲子时光

**目标**：在共读中建立正向因果链（缘位轴的核心场景）。

#### 4.7.1 共读流程

```
1. 家长选择"亲子时光"模式
2. 系统根据孩子的四维状态推荐1-3个主题
3. 家长和孩子一起阅读/讨论
4. 慧惠以"共读者"角色参与：
   - 不教"正确理解"
   - 陪伴两代人一起提问
   - 在对话断裂时轻轻衔接
5. 共读结束后：
   - 不评分
   - 慧惠问家长："今天你从孩子身上发现了什么？"
   - 慧惠问孩子："今天你想到了什么？"
6. 系统记录缘位数据（亲子因果链方向）
```

#### 4.7.2 家长陪伴策略推荐

基于孩子的四维状态，向家长推送"无辙迹"的陪伴建议：

| 孩子状态 | 家长常见误区 | 系统推荐 | 依据 |
|---|---|---|---|
| Oₜ低（节奏紊乱） | "你要专心！" | "试试在书桌上只放一本书" | 宇位减法→时位降熵 |
| Cₖ低（觉知低迷） | "你怎么这么差！" | "今天他在恐龙上花了5分钟，你注意到了吗？" | 根守护+兴趣子空间发现 |
| Kᵧ<0（恶性循环） | "你看看人家小禾！" | "今天试试不评价，只听他说" | 克——打断恶性因果链 |
| S接近θ_critical | （无意识忽视） | "他好像快'通了'，今天多给他一点安静的空间" | 化——临界点的守护 |

**关键**：家长端的所有推荐都必须是"建议"而非"指令"，且用词需避免引发焦虑。

### 4.8 FM-07：教师仪表盘

**目标**：为教师提供班级四维态势感知，实现"无弃人"。

#### 4.8.1 班级四维热力图

- X轴：时位（Oₜ），Y轴：宇位（Eᵤ），颜色：识位（Cₖ），节点大小：缘位（Kᵧ）
- 每个学生一个节点，聚合为班级散点图
- **不显示学生姓名**——仅显示匿名ID，教师点击后可查看脱敏画像

#### 4.8.2 "弃人"风险预警

| 预警级别 | 条件 | 系统行为 |
|---|---|---|
| 🟡 关注 | Cₖ持续3天<0.3 | 静默标记，不通知教师 |
| 🟠 提醒 | Cₖ持续7天<0.2 且 Kᵧ<0 | 推送脱敏干预建议至教师 |
| 🔴 守护 | Cₖ持续14天<0.15 且 Oₜ<0.2 且 Kᵧ<0 | 触发核心素养守护模式，教师端显示"优势子空间发现" |

**关键**：即使最高级别预警，也不使用"差生""问题学生"等标签。替代用语：
- ~~"差生"~~ → "优势子空间待发现"
- ~~"注意力缺陷"~~ → "时位节律重建中"
- ~~"不合作"~~ → "缘位因果链待正向闭合"

#### 4.8.3 "师-资"双向流转可视化

- 展示班级内的因果链网络
- 高亮"善缘"：谁帮助了谁、谁从谁那里获得了启发
- **不贵其师**：帮助者不获得特权标签
- **不爱其资**：被帮助者的数据不被工具化利用

### 10.1 术语表

| 术语 | 定义 |
|---|---|
| **道境空间坐标系** | 融合《道德经》哲学与现代科技的四维认知模型（时位·宇位·识位·缘位） |
| **时位轴** | 熵流脉动——行为节奏的有序/无序程度，输出Oₜ |
| **宇位轴** | 能量涟漪——资源/注意力的分布均衡度，输出Eᵤ |
| **识位轴** | 觉知光锥——认知负荷与觉知清晰度，输出Cₖ |
| **缘位轴** | 因果纠缠——人际/人机互动的正向因果链，输出Kᵧ |
| **存在度S** | 四维积分值，S≥θ_critical时质变发生 |
| **玄鉴引擎** | 四维融合决策引擎，内嵌伦理损失函数 |
| **无辙迹** | 干预不被用户感知为"被干预"，如春雨润物 |
| **核心素养守护模式** | Cₖ<0.2时自动进入的最低功耗陪伴模式 |
| **四季常青** | 根（自爱）·芽（自制）·叶（共情）·果（贡献）四组底线能力 |
| **兴趣子空间** | 全局四维低迷时，某个局部领域（如恐龙）的四维值异常高 |
| **成核生长** | 从兴趣子空间突破开始，逐步向全域迁移的成长模式 |
| **师-资流转** | 高识位者的觉知（师）为低识位者提供参照，低识位者的试错（资）为系统优化模型 |
