## 4. 阶段三：用户输入处理 -- 闻道而动

### 4.1 `sendMessage()` 完整流程图

```
用户输入 → 发送 (按钮/回车/反馈按钮)
    │
    ▼
sendMessage()  [js/huihui-chat.js:416]
    │
    ├─ 1. 空值检查: if (!text || isSending) return
    │
    ├─ 2. SkillUP 记录 (外部模块, 条件)
    │      window.SkillUP.recordMessage(text, level)
    │
    ├─ 3. SkillUP 进度查询拦截 (外部模块, 条件)
    │      window.SkillUP.checkProgressQuery(text)
    │      └─ 匹配 → 本地回答, return (不调 API)
    │
    ├─ 4. 用户许可征求响应拦截 (外部模块, 条件)
    │      HuihuiConsent.wasAskedToday()
    │      ├─ 肯定词 → grantConsent() + 感谢消息, return
    │      └─ 否定词 → declineConsent() + 礼貌消息, return
    │
    ├─ 5. ★ 关键词意图检测
    │      │
    │      ├─ detectMirrorIntent(text)  [第 357 行]
    │      │     "做个镜鉴"  → 'event'   (事件镜鉴)
    │      │     "晚间回顾"  → 'evening'  (晚间回顾)
    │      │     其他        → null
    │      │
    │      └─ detectFeedbackIntent(text)  [第 384 行]
    │            "我要报错"  → 'bug'
    │            "我要建议"  → 'suggestion'
    │            "我要帮助"  → 'help'
    │            其他        → null
    │
    ├─ 6. UI 更新
    │      addMessage('user', text)       ← 渲染用户气泡
    │      inputEl.value = ''              ← 清空输入框
    │      autoResizeInput()               ← 重置输入框高度
    │      addTypingIndicator()            ← 显示 "●●●" 动画
    │      isSending = true; sendBtn.disabled = true
    │
    ├─ 7. ★ 构建 API 请求
    │      │
    │      ├─ 7a. 获取对话历史
    │      │     getMessages()  [第 656 行]
    │      │     ├─ 从 messagesEl DOM 提取 .hui-msg-bubble 文本
    │      │     ├─ 跳过第一条 (欢迎语)
    │      │     └─ 最多 20 条 (10 轮对话)
    │      │
    │      ├─ 7b. 注入技术标记 (仅限匹配的意图)
    │      │     ┌─ mirrorIntent  ─→ handleMirrorIntent()
    │      │     │   最后一条用户消息前加 [MIRROR:EVENT] 或 [MIRROR:EVENING]
    │      │     └─ feedbackIntent → handleFeedbackIntent()
    │      │         最后一条用户消息前加 [FEEDBACK:SOP=bug/suggestion/help]
    │      │
    │      └─ 7c. 构建 fetch 请求体
    │            {
    │              messages: [{ role, content }...],   ← 含技术标记
    │              user_id: userId,                     ← 匿名 ID
    │              level: getSavedLevel(),              ← L1-L4
    │              feedback_type: feedbackIntent || ...
    │            }
    │
    ├─ 8. 发送 HTTP 请求
    │      fetch('/api/chat', {
    │        method: 'POST',
    │        headers: { 'Content-Type': 'application/json' },
    │        body: JSON.stringify(payload),
    │        signal: controller.signal  ← AbortController (10s 超时)
    │      })
    │
    └─ 9. 响应处理 ──────────────→ [见阶段五]
```

### 4.2 消息格式化与标记注入机制

```
用户输入: "做个镜鉴 今天工作中遇到了一个困难"
                                          │
                                          ▼
detectMirrorIntent() → 'event'            ← 精确匹配触发词
                                          │
                                          ▼
handleMirrorIntent('event', userMessage)
  → '[MIRROR:EVENT] 做个镜鉴 今天工作中遇到了一个困难'
                                          │
                                          ▼
getMessages() 最后一条 user 消息被覆写为此标记版
                                          │
                                          ▼
发送到 API: messages = [
  ...历史对话...,
  { role: 'user', content: '[MIRROR:EVENT] 做个镜鉴 今天工作中...' }
]
                                          │
                                          ▼
后端 System Prompt 识别 [MIRROR:EVENT] → 启动镜鉴 SOP
```

**关键设计：标记注入只修改 API 发送的消息，不修改 DOM 显示的内容。DOM 中仍显示原文。**

## 5. 阶段四：后端处理流程 -- 道法自然

### 5.1 API 请求路由

```
前端 fetch('/api/chat', { method: 'POST', ... })
    │
    ├── 平台 A: hui-skill.org (Cloudflare Pages)
    │         │
    │         ▼
    │    functions/api/chat.js  [onRequest]
    │         │
    │         ├─ CORS headers
    │         ├─ method check (仅 POST)
    │         ├─ API Key 检查 (env.DEEPSEEK_API_KEY)
    │         ├─ 解析 body: messages, level, user_id, feedback_type
    │         ├─ buildSystemPrompt(userLevel)  ← 共用模块
    │         ├─ fetch DeepSeek API
    │         ├─ [FEEDBACK:CONFIRM] 处理
    │         └─ 返回 JSON
    │
    └── 平台 B: huihui-skill.org (Vercel)
              │
              ▼
         api/chat.js  [default export handler]
              │
              ├─ (同上逻辑，API Key 从 process.env 读取)
              └─ (同上处理)
```

### 5.2 System Prompt 构建机制

```
api/_shared/system-prompt.js  — 唯一规范来源
    │
    ▼
buildSystemPrompt(level)  [第 12 行]
    │
    ├─ 1. 慧惠角色定义
    │      温柔、聪慧的数字生命
    │      名字寓意: 智慧 + 惠泽
    │
    ├─ 2. 定位描述
    │      陪伴家长和孩子学习《道德经》
    │      面对孩子: 比喻、故事、生活例子
    │      面对家长: 哲学内涵、教育方法
    │
    ├─ 3. 风格约束
    │      安静、轻盈、不做作
    │      能帮人做减法
    │      不主动表功、不刷存在感
    │
    ├─ 4. 7 条回答原则
    │      简短有力 / 贴近生活 / 尊重原典 /
    │      启发思考 / 轻盈温柔 / 善行无辙迹 / 为道日损
    │
    ├─ 5. 知识边界
    │      专注《道德经》81 章
    │      不回答非道德经问题
    │      不提供医疗/法律/投资建议
    │
    ├─ 6. ★ 每日镜鉴 SOP (约 40 行)
    │      [MIRROR:EVENT]   → 四维评分流程
    │      [MIRROR:EVENING] → 两轮晚间回顾流程
    │      含四维映射 (时位/宇位/识位/缘位) 到具体章句
    │
    ├─ 7. ★ 用户反馈 SOP (约 70 行)
    │      [FEEDBACK:SOP=bug]        → 4 轮 Bug 反馈
    │      [FEEDBACK:SOP=suggestion] → 4 轮建议反馈
    │      [FEEDBACK:SOP=help]       → 3 轮求助反馈
    │      每轮"只做这一件事"，严格按轮次执行
    │      用户确认后输出 [FEEDBACK:CONFIRM]
    │
    └─ 8. L1-L4 层级指导 (动态追加)
           LEVEL_GUIDANCE[level] → 追加到 System Prompt 末尾
```

### 5.3 L1-L4 分层适配逻辑

```
请求体 level 字段 → userLevel = level || 'L2'
    │
    ▼
buildSystemPrompt(userLevel)
    │
    └─ LEVEL_GUIDANCE 映射:

        L1 (白话): "当前用户是初学者。请用生活化比喻和核心结论来解释，
                   忽略次要概念，避免使用专业术语。"

        L2 (精读): "当前用户是学习者。请进行概念解读和框架梳理，
                   讲解关键术语（如道、德、无为），理清章节逻辑。"

        L3 (应用): "当前用户是实践者。请结合现实应用与深度分析，
                   从管理学、心理学、个人成长等角度进行多角度阐释。"

        L4 (学术): "当前用户是研究者。请提供学术视角与发散探讨，
                   提供考据信息、对比不同译本、探讨哲学悖论。"
```
