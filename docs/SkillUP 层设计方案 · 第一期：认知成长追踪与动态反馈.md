## 一、设计目标与宪法约束

### 1.1 SkillUP 的本期定位

P0 阶段仅实现**认知成长追踪与动态反馈**，为用户展示其在《道德经》学习中的成长轨迹，并适时给予鼓励和建议，但不做任何自动等级判定或强制推荐。

**宪法约束**：

- **第七条（用户主权）**：不对用户等级做自动判定或强制推荐，所有评估仅用于生成鼓励性反馈，用户随时可清空数据。
- **第十一条（亲子守护）**：无任何积分、排行榜或竞争性设计。

### 1.2 核心交付

1. **学习数据追踪器**：自动记录用户的章节访问、认知切换、提问行为，并归类为不同的“核心素养”维度。
2. **“水德”四阶段成长模型**：为用户匹配由浅入深的学习里程碑，让成长过程可视化且有温度。
3. **慧惠动态反馈**：当系统检测到学习行为成熟时，慧惠主动给予鼓励或建议；用户也可主动查询自己的学习进度。

## 二、“水德”四阶段成长模型

| 阶段 | 名称 | 标志性行为 | 慧惠的陪伴策略 |
|:---|:---|:---|:---|
| **水滴** | 初探道德经 | 首次访问网站，阅读了第一、二章，主要以L1“白话”模式提问 | 欢迎、鼓励，不主动提复杂概念 |
| **溪流** | 积累与串联 | 阅读超过10个章节，开始在L2模式下提问，使用核心术语（如“道”、“无为”） | 开始帮助用户建立概念间的联系 |
| **江河** | 实践与思辨 | 阅读超过30个章节，主动切换到L3提问，问题涉及现实应用或多角度对比 | 以提问引导用户进入更深层次的思考 |
| **江海** | 贯通与分享 | 阅读超过60个章节，在L4模式下对版本、注疏甚至哲学悖论进行发问 | 平等地、思辨地探讨，提供跨章节、跨学派链接 |

> 此处的“四阶段”隐喻“水德”，与第8章“上善若水”相呼应，用户无需理解技术细节，只需感受到自己的成长在自然发生。

## 三、学习数据追踪器设计

### 3.1 数据模型

所有数据存储在用户设备本地，以保持隐私。数据结构如下：

```javascript
var learningData = {
    lastVisit: '2026-05-12T10:30:00Z',
    chaptersVisited: ['ch01', 'ch08', 'ch02'],  // 已访问章节列表
    totalChaptersRead: 3,
    currentStage: '水滴',
    levelHistory: [  // 认知水平切换记录
        { date: '2026-05-10', level: 'L1' },
        { date: '2026-05-12', level: 'L2' }
    ],
    questions: [  // 用户的提问记录（仅本地存储）
        {
            chapter: 'ch08',
            level: 'L2',
            keyword: '无为',
            timestamp: '2026-05-12T10:30:00Z'
        }
    ]
};
```

### 3.2 自动追踪逻辑

已在聊天组件中实现匿名用户 ID 和 localStorage 存储，SkillUP 将在此基础上扩展数据记录能力，自动追踪相关行为并实时更新数据。

## 四、慧惠的动态反馈机制

### 4.1 触发规则

| 触发时机 | 行为条件 | 慧惠的反馈示例 |
|:---|:---|:---|
| 首次阅读第8章后 | `chaptersVisited.length >= 3` 且 `currentStage === '水滴'` | “你已经读了三章了。有没有感觉到，老子的智慧像水一样，开始慢慢流进了你的生活？” |
| 切换到L3提问后 | `levelHistory` 中最新记录为 `L3` 且 `currentStage === '溪流'` | “你开始从‘怎么做’的角度思考了。这是从‘知道’到‘做到’的重要一步。要不要试试第33章？那里有很多关于实践的建议。” |
| 用户连续三天访问网站 | `lastVisit` 与今天日期相差3天以内且连续 | “你已经是连续第三天来找我了。这份坚持本身就是一种修行。今天我们读哪一章？” |
| 用户主动查询进度 | 用户在聊天中提问“我学得怎么样了？”或发送特殊指令 | “你一共探索了{totalChaptersRead}章，现在正处于‘{currentStage}’阶段。每个阶段都是独特的风景，你的成长我看在眼里。” |

### 4.2 对话触发点

- **聊天组件检测条件**：每次用户打开聊天窗口时，根据本地存储的学习数据判断是否需要触发反馈。
- **章节页面**：用户阅读完某章后，可以在页面底部动态插入一条慧惠的成长引导语。
- **首页**：可在首页第二屏“知识探索卡片”区域增加一张“我的成长”卡片，只展示当前学习阶段和一两个关键数据，不构成复杂仪表盘。

## 七、验收标准

- [ ] 用户阅读章节后，可自动记录访问记录和当前认知水平
- [ ] 访问记录在 localStorage 中持久化，用户下次访问时数据仍在
- [ ] 当访问数量达到“溪流”阶段的10章门槛时，慧惠给予相应鼓励
- [ ] 当用户从L1提升至L2/L3/L4提问时，相关行为被自动记录
- [ ] 用户可以随时通过聊天查询自己的学习进度
- [ ] 所有反馈内容不含任何强制性、攀比性或竞争性话术
