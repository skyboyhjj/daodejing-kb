## 一、任务概述

在已有的学习数据追踪器基础上，实现完整的“水德四阶段”模型：根据用户的章节访问量、认知水平切换行为和提问频次，自动判定用户当前所处的成长阶段，并通过首页“我的成长”卡片和慧惠聊天窗口的轻量反馈，让用户自然感受到自己的成长。

### 核心交付

1. **阶段判定算法**：纯前端逻辑，根据 localStorage 中的学习数据自动计算当前阶段
2. **“我的成长”卡片**：在首页第二屏展示用户当前阶段与关键成长数据
3. **慧惠动态反馈**：当用户达到新阶段时，慧惠在聊天中给予温暖的肯定
4. **阶段转换纪念**：在章节页面中，通过轻量引导栏记录阶段跃迁时刻


## 二、阶段定义与判定标准

### 2.1 四阶段速览

| 阶段 | 名称 | 标志性特征 | 慧惠的陪伴策略 |
|:---|:---|:---|:---|
| **水滴** | 初探道德经 | 刚开始阅读，主要以L1模式提问 | 欢迎、鼓励，不主动提复杂概念 |
| **溪流** | 积累与串联 | 阅读一定数量章节，开始使用L2模式提问 | 帮助建立概念间的联系 |
| **江河** | 实践与思辨 | 大量章节阅读，主动切换到L3提问 | 以提问引导更深层次思考 |
| **江海** | 贯通与分享 | 覆盖绝大多数章节，在L4模式下进行学术探讨 | 平等地、思辨地探讨，提供跨章节链接 |

### 2.2 判定算法

```javascript
function calculateStage(learningData) {
    var chaptersRead = (learningData.chaptersVisited || []).length;
    var hasL2 = (learningData.levelHistory || []).some(function(h) { return h.level === 'L2'; });
    var hasL3 = (learningData.levelHistory || []).some(function(h) { return h.level === 'L3'; });
    var hasL4 = (learningData.levelHistory || []).some(function(h) { return h.level === 'L4'; });
    var questionCount = (learningData.questions || []).length;

    // 江海：覆盖 ≥60 章，且使用过 L4 模式，提问 ≥10 次
    if (chaptersRead >= 60 && hasL4 && questionCount >= 10) {
        return '江海';
    }
    // 江河：覆盖 ≥30 章，且使用过 L3 模式，提问 ≥5 次
    if (chaptersRead >= 30 && hasL3 && questionCount >= 5) {
        return '江河';
    }
    // 溪流：覆盖 ≥10 章，且使用过 L2 模式
    if (chaptersRead >= 10 && hasL2) {
        return '溪流';
    }
    // 水滴：默认阶段
    return '水滴';
}
```

**算法特点**：每个更高阶段的条件**包含**前一阶段的条件，确保用户不会因为单次行为波动而在阶段间反复横跳。阶段只升不降，一旦达到某阶段，即使后续行为减少，也不会降级——这是“她记得你的成长”。


## 三、“我的成长”卡片设计

### 3.1 卡片位置与触发条件

- **位置**：首页第二屏，六张知识探索卡片下方，独立一张卡片
- **触发条件**：用户已产生至少一条学习数据（如访问过任一章节）。若用户从未使用过学习功能，卡片不展示
- **交互**：点击卡片可展开学习详情，再次点击收起

### 3.2 卡片内容（各阶段统一模板）

```
┌──────────────────────────────────────────────┐
│  🪷 我的修行之旅                               │
│                                              │
│  你从一颗水滴出发，现在是潺潺的溪流了。           │
│                                              │
│  📖 读过了 15 章                              │
│  💬 和慧惠聊过 8 次                            │
│  🎯 开始用“精读”模式探索了                      │
│  🗓️ 最近在读：第16章 · 致虚守静                 │
│                                              │
│  继续前行吧，江河不远了。                        │
│  [查看详情]                                    │
└──────────────────────────────────────────────┘
```

### 3.3 阶段特定的卡片文案

| 阶段 | 卡片标题 | 结尾引导语 |
|:---|:---|:---|
| 水滴 | 你从一颗水滴出发。 | 每一章都是一次新的遇见。 |
| 溪流 | 你从一颗水滴出发，现在是潺潺的溪流了。 | 继续前行吧，江河不远了。 |

```
   · 2026-05-30 溪流→江河（读完30章，开始实践应用）
```


## 四、慧惠动态反馈机制

### 4.1 阶段跃迁反馈

当用户行为触发了阶段跃迁（如从溪流→江河）时，**不弹窗、不推送通知**，而是在用户下一次打开聊天窗口时，慧惠在欢迎语中融入阶段肯定的内容：

**示例（溪流→江河）**：
> “你好呀！今天想从哪一章开始？——顺带说一句，你已经读了30章了。从溪流汇成江河，你知道这意味着什么吗？老子说‘江海所以能为百谷王者，以其善下之’。你读得越多，越谦卑。这就是成长。”

### 4.2 章节末尾的引导（可选，P2实施）

当用户阅读章节数量达到阶段门槛（如第10章、第30章、第60章）时，在章节末尾引导栏中增加一句轻量提醒：

```
🌿 慧惠说：你已经读完了第10章。这是从水滴汇成溪流的时刻。要不要记录一下今天的感悟？
```

### 4.3 阶段跃迁纪念（可选，P2实施）

在阶段跃迁发生时，在“我的成长”卡片中增加一条阶段跃迁记录，用户可在详情中查看，但不主动推送。
