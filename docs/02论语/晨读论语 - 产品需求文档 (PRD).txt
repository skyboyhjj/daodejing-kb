晨读论语 - 产品需求文档 (PRD)
￼
一、项目概述
1.1 基本信息
• 应用名称：晨读论语
• 网站类型：品牌展示型（文化传播博客）
• 核心定位：每日清晨传播论语智慧的传统文化博客，以温暖的问候和深度的经典解读陪伴用户开启美好一天
• 目标用户：家人、朋友、传统文化爱好者
1.2 技术配置
• 数据库：需要数据库（暂不启用，待页面生成后启用）— 用于存储论语文章数据、分类标签、订阅用户信息
• 网站主语言：中文
• 响应式要求：适配移动端/平板/桌面（参考小米早安画报的移动端优先设计）
￼
二、核心功能
2.1 MVP功能（高优先级）
2.1.1 每日论语推送
每天自动更新论语内容，包含早安问候、日期信息、论语原文、参考翻译和现代启示
2.1.2 文章列表展示
按日期倒序展示所有论语文章，支持浏览历史内容，卡片式布局展示标题、日期和摘要
2.1.3 分类标签
按论语篇章分类（如学而篇、为政篇、先进篇等），用户可按分类筛选查看相关文章
2.2 增强功能（中优先级）
2.2.1 订阅功能
用户可输入邮箱订阅每日推送提醒，新文章发布时发送邮件通知
2.2.2 AI助手解读
提供AI助手帮助用户深度解读论语内容，回答用户关于论语的疑问
￼
三、页面结构
3.1 首页布局
整体布局模式：移动端优先的单列沉浸式滚动布局，参考小米早安画报的视觉风格
核心区域：
1. Header（顶部导航）
◦ 左侧：Logo「晨读论语」
◦ 右侧：导航菜单（首页、论语列表、分类、订阅）
◦ 移动端：汉堡菜单
2. Hero区（首屏）⚠️ 关键视觉区域
背景实现方式：采用单张大图背景 + 渐变色叠加
◦ 图片描述：清晨阳光、日出、自然风景或中国传统水墨风格的背景图，传递温暖、宁静、充满希望的感觉
◦ 前景内容：
■ 核心价值主张：「晨读论语 · 每日智慧」
■ 简短说明：「每天清晨，与经典同行」
■ 行动按钮：「阅读今日论语」
◦ 设计意图：通过温暖的晨光背景和精炼的文案，让用户立刻感受到宁静、温暖和文化的力量
3. 今日论语区（核心内容）
◦ 早安问候语（如：「早安\[太阳\] 感恩美好的一天开始了\[爱心\]」）
◦ 日期信息（阳历 + 农历）
◦ 论语篇章标题（如：《论语·先进第十一》）
◦ 论语原文（带章节编号）
◦ 参考翻译
◦ 词语注释
◦ 现代启示
◦ 分享按钮
4. 历史文章区
◦ 标题：「往期论语」
◦ 卡片列表展示（3-5篇最新文章）
◦ 每篇卡片：日期、篇章标题、摘要
◦ 「查看更多」按钮
5. 分类导航区
◦ 标题：「按篇章浏览」
◦ 标签云形式展示论语各篇章（学而篇、为政篇、八佾篇等）
◦ 点击跳转到对应分类的文章列表
6. Footer（底部信息）
◦ 联系方式、社交媒体链接、版权信息
3.2 关键内页
页面：论语列表页
页面作用：用户浏览所有历史论语文章，按日期查找内容
主要内容区域：
1. 页面标题：「论语大全」
2. 筛选器：按篇章分类筛选
3. 文章列表：卡片式布局，每张卡片包含日期、篇章标题、原文摘要
4. 分页导航
转化设计：每篇卡片可点击进入详情页，列表页底部放置「订阅每日推送」按钮
页面：分类详情页
页面作用：展示特定篇章（如「先进篇」）下的所有文章
主要内容区域：
1. 分类标题：「论语·先进第十一」
2. 分类简介：简要介绍该篇章的主题和背景
3. 文章列表：该篇章下的所有文章（按日期倒序）
4. 返回全部分类的链接
转化设计：列表底部放置「订阅该分类更新」按钮
页面：订阅页
页面作用：用户输入邮箱订阅每日论语推送
主要内容区域：
1. 页面标题：「订阅每日论语」
2. 价值说明：「每天清晨，论语智慧准时送达您的邮箱」
3. 订阅表单：邮箱输入框 + 订阅按钮
4. 隐私说明：「我们尊重您的隐私，不会泄露您的邮箱」
转化设计：表单提交后显示成功提示，引导用户查看往期内容
页面：AI助手页
页面作用：用户与AI助手互动，深度解读论语内容
主要内容区域：
1. 页面标题：「AI助手慧惠」
2. 助手介绍：「你好呀！我是慧惠。关于《论语》，有什么想聊的？」
3. 对话界面：聊天窗口 + 输入框
4. 常见问题推荐：「孔子的人才观是什么？」「如何理解’己所不欲勿施于人’？」
转化设计：对话中可推荐相关论语文章，引导用户深入阅读
￼
四、核心交互流程
4.1 流程：用户订阅流程
操作步骤：
1. 用户点击「订阅」按钮或进入订阅页
2. 输入邮箱地址
3. 点击「订阅」按钮 → 显示loading
4. 成功：显示「订阅成功！感谢您的关注」，发送确认邮件
5. 失败：显示错误信息（如邮箱格式错误），允许重试
异常处理：网络超时/服务器错误时显示「订阅暂时不可用，请稍后再试」
4.2 流程：AI助手问答流程
操作步骤：
1. 用户在AI助手页输入问题
2. 点击发送 → 显示loading动画
3. AI返回解读内容
4. 用户可继续追问或查看相关推荐文章
异常处理：AI服务不可用时显示「AI助手慧惠暂时休息中，您可以先浏览往期论语」

五、视觉与品牌策略
5.1 品牌个性定位
核心定位：温暖亲和型 + 文化传承型
目标情感：
• 第一层感受：温暖、宁静、充满希望
• 第二层感受：文化认同、智慧启迪、心灵滋养
差异化策略：
• 我们不是严肃的学术网站，而是温暖的晨间陪伴者
• 核心差异点：以「早安问候 + 论语智慧」的形式，让传统文化融入现代人的日常生活
设计决策摘要：
• 主色：暖橙色系 #FF9A56 | 辅色：米白色 #F5F0E6 | 强调色：中国红 #C41E3A
• 布局模式：移动端优先的单列沉浸式滚动 + 卡片式内容展示
• 动效基调：柔和、温暖、流畅
• 字体策略：衬线字体传递文化感 + 清晰易读的正文
• 采用趋势：精致微交互 + Scroll-driven淡入动画
5.2 设计语言
整体风格：温暖人文风，参考小米早安画报的清新、温暖、有质感的视觉风格
核心调性：温暖、宁静、文化感、亲和力
设计目标：让用户在清晨打开网站时感受到温暖的问候和文化的力量，开启美好的一天
5.3 色彩方案
主背景色：米白色 #F5F0E6
• 应用：页面背景、卡片背景
主色调：暖橙色 #FF9A56
• 应用：标题文字、按钮、重要图标
强调色：中国红 #C41E3A
• 应用：CTA按钮、重点标注、特殊节日装饰
辅助色：
• 深灰色 #333333：正文文字
• 浅灰色 #999999：辅助文字、日期信息
• 金色 #D4AF37：点缀装饰（如分隔线、图标）
文字色：
• 深色背景时：白色 #FFFFFF
• 浅色背景时：深灰色 #333333
图片/图标风格：
• 图片：清晨阳光、自然风光、中国传统水墨风格插画
• 图标：简洁线条图标，略带中国风元素（如卷轴、毛笔、印章等）
• Emoji使用：适度使用\[太阳\]\[爱心\]\[玫瑰\]等表情符号增加亲和力
5.4 布局策略
首屏布局：全屏Hero区，背景大图 + 居中文字
内容区布局：单列垂直布局（移动端），桌面端可适当增加侧边栏
留白策略：平衡留白，内容区有足够呼吸感但不过度稀疏
排版原则：
• 大标题传递文化感和力量感
• 论语原文使用稍大字号突出重要性
• 翻译和注释使用清晰易读的字号
• 适当使用分隔线和留白区分不同内容块
5.5 字体规范
标题字体：
• 样式：衬线体（如思源宋体、Noto Serif SC）传递文化感
• 大小：H1 48-56px / H2 36px / H3 28px
正文字体：
• 样式：无衬线体（如思源黑体、Noto Sans SC）确保易读性
• 大小：16-18px
论语原文：
• 样式：衬线体加粗，突出经典感
• 大小：20-22px
小字/辅助文字：
• 大小：14px（日期、注释等）
5.6 交互动效
整体基调：柔和、温暖、流畅
核心动效：
1. Scroll-driven淡入动画：内容区块随滚动依次淡入，营造沉浸阅读体验
2. 精致微交互：按钮hover时轻微放大（scale 1.02-1.05）+ 阴影加深
3. 卡片悬浮效果：文章卡片hover时轻微上浮（translateY -4px）+ 阴影变化
4. 平滑页面过渡：页面切换时淡入淡出效果
动效参数规范：
• 过渡时长：快速反馈250ms，内容切换400ms，页面级过渡600ms
• 缓动函数：cubic-bezier(0.4, 0, 0.2, 1)（克制优雅）
• 滚动触发：元素进入视口15%时开始动画，translateY(20px→0) + opacity(0→1)
• hover效果：位移幅度4px，缩放范围1.02-1.05
• 交错延迟：列表项依次进入时每项间隔80ms
动效性能原则：
• 移动端简化动效，优先考虑性能
• 避免过度动画干扰阅读体验
• 关键转化路径上的动效不能影响用户操作
5.7 品牌元素
Logo使用：导航栏左侧、页脚、加载动画
品牌视觉符号：
• 中国传统元素：卷轴、印章、水墨笔触作为装饰
• 分隔线：可使用简化的传统纹样
• 加载动画：旋转的书卷或日出动画
品牌色彩系统：确保所有页面色彩统一，暖色调贯穿全站
5.8 文案调性（含示例）
语言风格：温暖、亲切、有文化底蕴但不晦涩
具体示例：
• 标题文案：
◦ 「晨读论语 · 每日智慧」
◦ 「每天清晨，与经典同行」
◦ 「论语相伴，心安每一天」
• 正文文案：
◦ 「早安\[太阳\] 感恩美好的一天开始了\[爱心\]」
◦ 「今天，让我们一起学习《论语·先进第十一》」
◦ 「孔子的智慧，穿越千年依然照亮我们的生活」
• CTA文案：
◦ 「阅读今日论语」
◦ 「订阅每日推送」
◦ 「向AI助手提问」
禁止：
• ❌ 过于学术化、晦涩难懂的表达
• ❌ 冷冰冰的官方口吻
• ❌ 过度营销化的语言

六、多端适配策略
6.1 响应式要求
网站必须在移动端/平板/桌面端提供良好体验，优先保证移动端的阅读体验（参考小米早安画报的移动端设计）
6.2 断点定义
设备类型
屏幕宽度
布局特征
移动端
< 768px
单列，导航折叠为汉堡菜单
平板
768px - 1023px
单列或双列，导航完整展示
桌面端
≥ 1024px
双列布局（内容区 + 侧边栏），完整导航
6.3 核心适配规则
导航栏：
• 移动端：汉堡菜单，点击展开
• 桌面端：完整水平菜单
Hero区：
• 移动端：全宽背景图，标题40-48px
• 桌面端：全宽背景图，标题56-64px
内容卡片：
• 移动端：单列，卡片宽度100%
• 平板：单列或双列
• 桌面端：列表页可双列展示
论语详情内容：
• 移动端：单列垂直，原文和翻译分段清晰
• 桌面端：可适当增加左右分栏（原文左，翻译右）
性能优化：
• 移动端：图片懒加载，简化动效，压缩图片
• 桌面端：高分辨率图片，丰富交互

七、补充说明
内容更新机制：
• 每日论语内容需手动或通过自动化脚本更新
• 平台将根据数据库表自动生成管理后台（增删改查），开启数据库后，用户可在「管理 → 内容管理」中操作。无需在前端开发任何管理功能
AI助手说明：
• AI助手功能需要集成AI API，提供论语相关的问答服务
• 可预设常见问题库，提升回答质量
订阅功能说明：
• 订阅功能需要集成邮件发送服务
• 新文章发布时自动触发邮件通知
