**任务：修复移动端聊天键盘遮挡回复内容的问题**

**问题描述：**
在手机端访问网站 https://hui-skill.org/ 的“亲子时光”与“问慧惠”聊天功能时，慧惠回复消息后输入框立即自动聚焦并弹出键盘，导致最新的回复内容被键盘遮挡，用户无法看到回复就进入输入状态。

**解决方案：**
在回复内容渲染完成后，先执行滚动到底部（`scrollToBottom()`），然后将输入框的 `focus()` 操作延迟约 400 毫秒执行，确保用户能完整阅读回复后再进行输入。

**涉及文件：**
1. `js/family.js`
2. `js/huihui-chat.js`

**具体修改内容：**

### 一、`js/family.js`

#### 修改点 1：`sendNextRound` 函数内部（约第 309–317 行）
将：
```javascript
setInputEnabled(true);
userInputEl.focus();
// 检测是否应该结束...
```
改为：
```javascript
setInputEnabled(true);
scrollToBottom();
setTimeout(function () {
  userInputEl.focus();
}, 400);
// 检测是否应该结束...
```

#### 修改点 2：`startChapter` 函数内部（约第 307–309 行，许可征求回调中）
将：
```javascript

userInputEl.focus();
```
改为：
```javascript
setInputEnabled(true);
scrollToBottom();
setTimeout(function () {
  userInputEl.focus();
}, 400);
```

### 二、`js/huihui-chat.js`

#### 修改点 3：面板打开时的聚焦（约第 126 行）
将：
```javascript

scrollToBottom();
```
改为：


**注意事项：**
- 两个文件中均已存在 `scrollToBottom` 函数，无需新增定义。
- 延迟时间 400 毫秒为建议值，可根据实际体验微调（范围 300~600ms）。
- 请确保修改后不影响桌面端体验，且不改变其他业务逻辑。

**期望效果：**
- 手机端聊天时，慧惠回复后，聊天区域自动滚动至最新消息可见。
- 短暂延迟后输入框获取焦点，此时键盘弹出不会遮挡回复内容。
- 用户阅读流畅，对话体验自然。
