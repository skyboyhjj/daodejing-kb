# 修复移动端聊天键盘遮挡回复内容的问题

> 日期：2026-05-27 | 状态：设计方案 | 影响范围：`js/huihui-chat.js`、`_prod-check3.css`（需同步到 `css/` 主样式表）

---

## 1. 问题分析

### 1.1 现象

在移动端（iOS Safari / Android Chrome）打开慧惠聊天面板后，虚拟键盘弹起时，聊天面板的最底部（最新回复内容 + 输入框区域）被键盘遮挡，用户看不到 AI 的最新回复。

### 1.2 触发流程

```
用户点击聊天按钮 → openPanel() → chatPanel 全屏打开 (↓)
  → inputEl.focus()（立即触发键盘弹出）(↓)
  → scrollToBottom()（requestAnimationFrame 回调）(↓)
  → 键盘动画完成（~350-400ms 后）(↓)
  → 视口高度骤减 → 最新消息被键盘遮挡
```

### 1.3 根因分析

| 层次 | 问题 | 影响 |
|:---|:---|:---|
| **时序错位** | `scrollToBottom()` 在键盘动画**之前**执行。此时 `100vh` 仍是完整屏幕高度，`scrollTop` 被设置到"假底部"——当键盘弹出后视口缩小，实际可见底部被推到键盘后方 | 最新 AI 回复完全不可见 |
| **固定高度** | 移动端面板使用 `height: 100%` + `max-height: 100vh`，不随键盘动态调整 | 面板内容区域不收缩，消息区底部被键盘遮盖 |
| **缺少动态检测** | 没有监听键盘弹起/收起事件，无法在键盘动画完成后重新校准滚动位置 | 用户需要手动上滑查看最新回复 |
| **iOS Safari 特有问题** | iOS Safari 在 `input.focus()` 后延迟 300-400ms 才调整 `visualViewport`，期间任何基于 `innerHeight` 的计算都是错误的 | 在 iOS 上问题更严重 |
| **`requestAnimationFrame` 不足** | 当前 `scrollToBottom()` 仅请求一帧后就执行 `scrollTop = scrollHeight`，此时键盘尚未开始动画 | 键盘动画约需 15-30 帧 |

### 1.4 设备差异

| 平台 | 键盘行为 | 影响程度 |
|:---|:---|:---|
| **iOS Safari (≤ 15.x)** | 键盘以 overlay 方式弹出，不改变 `window.innerHeight`，但改变 `visualViewport.height` | **重** — 依赖 `innerHeight` 的方案完全失效 |
| **iOS Safari (≥ 16.x)** | 键盘以 resize 方式改变布局视口，`innerHeight` 和 `visualViewport.height` 同步变化 | **中** — 可检测但动画滞后 |
| **Android Chrome** | 键盘弹出时立刻 resize 视口，`window.innerHeight` 同步减小 | **中** — 变化快但 scroll 时序仍错位 |
| **Android 第三方输入法** | 部分输入法先弹键盘再 resize，时间差可达 500ms+ | **重** — 不可预测的延迟 |

## 2. 现有方案的问题（"400ms 延迟"的缺陷）

### 2.1 当前隐式等待机制

目前代码中没有显式的 400ms 延迟。实际上，问题出在 `scrollToBottom()` 通过 `requestAnimationFrame` 执行时，键盘尚未开始动���。常见的修复方案是增加 `setTimeout(fn, 400)` 延迟，但这存在以下缺陷：

### 2.2 固定延迟的五大问题

| 问题 | 描述 |
|:---|:---|
| **① 假设统一** | 假设所有设备的键盘动画都是 400ms。实际 iOS 约 350ms，Android 约 200-300ms，第三方输入法可达 500ms+ |
| **② 时间浪费** | 在快设备上用户需要额外等待不必要的时间，体验粘滞 |
| **③ 不够长** | 在慢设备 / 低端机上 400ms 仍不够，键盘遮挡依旧发生 |
| **④ 干扰其他操作** | 如果用户在 400ms 内手动滚动，延迟执行会覆盖用户的手动位置 |
| **⑤ 无重检机制** | 键盘弹出后视口稳定了，但没有二次校验重新滚动 |

## 3. 优化方案：基于 Visual Viewport API 的动态检测

### 3.1 核心思路

使用 **Visual Viewport API** (`window.visualViewport`) 监听视口高度的实时变化，当检测到视口高度稳定（键盘动画结束）时，触发精确的 `scrollToBottom()`。

### 3.2 技术选型对比

| 方案 | 原理 | 精度 | 兼容性 | 推荐 |
|:---|:---|:---|:---|:---|
| `setTimeout(fn, 400)` | 固定延迟 | 低 | 100% | ❌ |
| `window.addEventListener('resize', fn)` | 监听窗口 resize | 中（iOS 15- 不触发） | ~80% | ❌ |
| `visualViewport.addEventListener('resize', fn)` | 监听可视视口变化 | **高** | 94%+（iOS 13+, Android 5+） | ✅ |
| `IntersectionObserver` + 哨兵元素 | 检测底部哨兵是否可见 | 中 | 95%+ | 备选 |
| `input.addEventListener('focusin', fn)` + 防抖 | 聚焦触发 + 防抖 | 低 | 100% | 仅兜底 |

**推荐主方案**：Visual Viewport API + 稳定性检测 + `scrollIntoView`  
**兜底方案**：`setTimeout(fn, 400)` 作为最后 fallback

### 3.3 方案流程图

```
inputEl.focus() 触发键盘
      │
      ▼
window.visualViewport 开始改变
      │
      ▼
visualViewport resize 事件持续触发（键盘动画期间）
      │
      ▼ （防抖：距上次事件 ≥ 100ms 认为稳定）
视口高度连续 2 帧不再变化
      │
      ▼
键盘动画完成 → 触发 onKeyboardStable()
      │
      ├── 计算可见底部偏移
      ├── messagesEl.scrollTo({ top: messagesEl.scrollHeight, behavior: 'smooth' })
      └── 可选：调整面板 CSS 变量 --panel-visible-height
```

## 4. 具体实现代码

### 4.1 在 `js/huihui-chat.js` 中新增键盘管理模块

在 IIFE 内部、`scrollToBottom` 函数附近（约第 642 行前）插入以下代码：

```javascript
// ===== 移动端键盘管理（Visual Viewport API）=====
var keyboardStableTimer = null;
var lastViewportHeight = 0;
var keyboardFrameCount = 0;
var KEYBOARD_STABLE_FRAMES = 2;   // 连续稳定帧数
var KEYBOARD_STABLE_MS = 100;     // 防抖间隔 (ms)

/**
 * 检测是否为移动端
 */
function isMobile() {
    return window.innerWidth <= 480;
}

/**
 * 处理键盘稳定后的回调 — 重新滚动到底部
 */
function onKeyboardStable() {
    if (!isOpen || !isMobile()) return;

    var currentHeight = window.visualViewport
        ? window.visualViewport.height
        : window.innerHeight;

    // 计算面板应该可见的高度，调整 CSS 变量
    if (chatPanel && window.visualViewport) {
        var offsetTop = chatPanel.getBoundingClientRect().top;
        var visibleHeight = window.visualViewport.height - offsetTop;
        chatPanel.style.setProperty('--keyboard-visible-height', visibleHeight + 'px');
    }

    // 平滑滚动到面板消息区的最新消息
    if (messagesEl) {
        // 使用 scrollTo 替代直接设置 scrollTop，确保键盘弹出后视口已稳定
        messagesEl.scrollTo({
            top: messagesEl.scrollHeight,
            behavior: 'smooth'
        });
    }
}

/**
 * visualViewport resize 事件处理（带防抖）
 */
function handleViewportResize() {
    if (!isMobile()) return;

    var currentHeight = window.visualViewport
        ? window.visualViewport.height
        : window.innerHeight;

    // 高度未变化，跳过
    if (currentHeight === lastViewportHeight) return;
    lastViewportHeight = currentHeight;

    // 清除之前的稳定检测计时器
    if (keyboardStableTimer) {
        clearTimeout(keyboardStableTimer);
    }

    // 重置帧计数，开始新一轮稳定检测
    keyboardFrameCount = 0;
    checkViewportStable(currentHeight);
}

/**
 * 通过 requestAnimationFrame 检测视口高度是否连续 N 帧不变
 */
function checkViewportStable(targetHeight) {
    if (!isMobile() || !isOpen) {
        keyboardFrameCount = 0;
        return;
    }

    var currentHeight = window.visualViewport
        ? window.visualViewport.height
        : window.innerHeight;

    if (currentHeight === targetHeight) {
        keyboardFrameCount++;
        if (keyboardFrameCount >= KEYBOARD_STABLE_FRAMES) {
            // 视口已稳定，键盘动画完成
            onKeyboardStable();
            return;
        }
    } else {
        keyboardFrameCount = 0;
        targetHeight = currentHeight;
    }

    requestAnimationFrame(function () {
        checkViewportStable(targetHeight);
    });
}

/**
 * 初始化键盘监听
 */
function initKeyboardListener() {
    if (window.visualViewport) {
        window.visualViewport.addEventListener('resize', handleViewportResize);
        window.visualViewport.addEventListener('scroll', handleViewportResize);
    } else {
        // 兜底：监听 window resize + input focus 事件
        window.addEventListener('resize', function () {
            clearTimeout(keyboardStableTimer);
            keyboardStableTimer = setTimeout(onKeyboardStable, 400);
        });
    }
}
```

### 4.2 修改 `scrollToBottom()` 函数

将原函数改为：

```javascript
/** 滚动到底部（桌面端：立即；移动端：由键盘管理模块接管） */
function scrollToBottom() {
    if (isMobile()) {
        // 移动端：延迟一帧后尝试滚动，键盘弹起后的精确定位由键盘管理模块处理
        requestAnimationFrame(function () {
            if (messagesEl) {
                messagesEl.scrollTop = messagesEl.scrollHeight;
            }
        });
        return;
    }
    // 桌面端：立即执行
    requestAnimationFrame(function () {
        if (messagesEl) {
            messagesEl.scrollTop = messagesEl.scrollHeight;
        }
    });
}
```

在 `openPanel()` 的末尾（约第 301 行之前，`panelJustOpened = false` 的 setTimeout 之后）加入初始化调用。同时将原来的 `inputEl.focus()` 延迟到滚动之后：

```javascript
// 现有代码（第 226-236 行）修改为：
chatPanel.classList.add('open');
```

```css
/* ===== 移动端键盘适配（Visual Viewport 动态调整）===== */
#huihui-chat-panel {

    height: 100dvh;

    /* 支持键盘管理模块动态覆盖 */

/* 当 keyboard-visible-height 被 JS 设置时，覆写面板高度 */
#huihui-chat-panel.keyboard-visible {
    height: var(--keyboard-visible-height);
    max-height: var(--keyboard-visible-height);
}

/* 消息区域在键盘弹起时需要确保弹性 */
.hui-messages {
    /* 确保消息区在键盘适配时能正常滚动 */
    -webkit-overflow-scrolling: touch;
}
```

### 5.2 使用 `dvh` 单位替代 `vh`

| 属性 | 当前值 | 建议值 | 原因 |
|:---|:---|:---|:---|
| `#huihui-chat-panel` height | `100%` | `100dvh` | `dvh` 随键盘动态变化，iOS Safari 15.4+ 支持 |
| `#huihui-chat-panel` max-height | `100vh` | `100dvh` | 同上 |

**兼容性说明**：
- `100dvh` 在 iOS Safari ≥ 15.4、Chrome ≥ 108 支持
- 对于不支持的旧浏览器，保持 `100%` 作为 fallback：
  ```css
  height: 100%;
  height: 100dvh;  /* 覆盖前者 */
  ```

## 6. 兼容性处理矩阵

### 6.1 浏览器支持检测

```javascript
var supportsVisualViewport = 'visualViewport' in window;
var supportsDVH = CSS.supports('height', '100dvh');

// 策略选择
if (supportsVisualViewport) {
    // ★ 最优：Visual Viewport API + dynaviewport
    initKeyboardListener();
    useDynamicHeight();
} else if (supportsDVH) {
    // ★★ 中等：dvh 单位 + 固定延迟 fallback
    useDVHFallback();
} else {
    // ★★★ 兜底：400ms setTimeout + window resize
    useResizeFallback();
}
```

### 6.2 iOS / Android 差异处理

| 场景 | iOS Safari | Android Chrome | 处理方式 |
|:---|:---|:---|:---|
| 键盘弹出 | visualViewport resize 触发 3-5 次 | 触发 2-3 次 | 防抖 + 稳定帧检测覆盖两者 |
| 键盘收起 | visualViewport resize 同样触发 | 同左 | `scrollToBottom()` 校准 |
| 第三方输入法 | 部分输入法不触发 resize | 延迟可能 >500ms | 在 `focus` 事件 800ms 后做最终校准 |
| 横屏键盘 | visualViewport 大幅变化 | 同左 | 面板 `max-height` 限制确保可见 |

### 6.3 覆盖键盘收起的滚动校准

在 `closePanel()` 和 `sendMessage()` 的响应回调中，移动端面板关闭后的背景页可能因 iOS Safari 的橡皮筋效果产生异常偏移。在关闭后立即重置：

```javascript
// closePanel() 已在现有代码中处理: document.body.style.overflow = ''
// 额外增加 iOS Safari 修复：
if (window.visualViewport) {
    window.scrollTo(0, 0);  // 防止 iOS 键盘收起后的白底残留
}
```

## 8. 实施注意事项

### 8.1 防止 requestAnimationFrame 无限循环

在 `checkViewportStable()` 中加入硬性上限：

```javascript
var MAX_KEYBOARD_FRAMES = 60;  // 1 秒 @ 60fps

function checkViewportStable(targetHeight) {
    if (!isMobile() || !isOpen || keyboardFrameCount > MAX_KEYBOARD_FRAMES) {
        keyboardFrameCount = 0;
        return;
    }
    // ... 其余逻辑 ...
}
```

### 8.2 CSS 过渡避免闪烁

`transition: height 0.15s ease-out` 的 150ms 过渡时间经过权衡：
- 太短（< 100ms）：键盘变化期间面板频繁调整，产生抖动
- 太长（> 250ms）：用户感知到面板"跟不上"键盘

### 8.3 `_prod-check2.css` 与 `_prod-check3.css` 同步

两个 CSS 文件当前内容相同，但生产环境可能引用其中之一。**修改时需同时更新两个文件**，或确认哪个文件是活跃的生产版本后仅更新一个。

### 8.4 AGENTS.md 中的相关约束

根据项目宪法：
- **第十条（外挂增强）**：本次修改为 `huihui-chat.js` 内部新增模块，不修改网站核心功能
- **第八条（善行无辙迹）**：键盘适配在后台静默进行，用户无感知
- **第五条（减法优先）**：方案不新增 UI 元素，不增加用户操作步骤
