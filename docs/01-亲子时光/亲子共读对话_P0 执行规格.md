## 一、任务概述

基于《“亲子时光”功能 · 完整设计方案》，实现 P0 阶段的最小可用版本：由慧惠引导的亲子共读对话体验。

**核心交付**：
1. 亲子共读专属页面（`family.html`），可以从首页“亲子时光”卡片直达
2. 慧惠开场白（自适应3个年龄段）
3. 首期3章示例内容（第8章·上善若水、第1章·道可道非常道、第2章·天下皆知美之为美）
4. 简洁的对话式交互界面


## 二、功能范围

**P0 确定要做**：
- 亲子共读专属页面，包含慧惠形象 + 对话区 + 输入区
- 首次进入时，慧惠询问孩子年龄（用于后续自适应语言难度）
- 基于3个预设章节的亲子引导对话模板
- 每轮对话节奏控制（一个话题 → 一个提问 → 等待回应）
- 慧惠以亲子陪伴者身份对话（非老师、非测试官）

**P0 明确不做**：
- 不是亲子互动题库（P1）
- 不是育儿锦囊（P2）
- 不需要用户注册或登录
- 不持久化存储孩子的回答
- 不评估孩子的理解水平
- 不提供多子女画像切换

## 四、对话模板设计

### 4.1 年龄段自适应

| 年龄段 | 语言特点 | 对话篇幅 | 互动方式 |
|:---|:---|:---|:---|
| **4-6岁** | 用大自然、小动物、日常生活比喻 | 每次2-3句话 | 学动物叫、用手指、画下来 |
| **7-9岁** | 用故事、角色扮演、简单类比 | 每次3-4句话 | 说出自己的故事、角色扮演 |
| **10-12岁** | 引入更多原文、探讨更抽象的概念 | 每次4-5句话 | 提出自己的理解、与生活关联 |

### 4.2 三章示例内容

**第一章示例：第8章 · 上善若水**

核心观点（慧惠内化，不直接输出）：最高的善像水一样，利万物而不争。

4-6岁对话示例：

> **慧惠**：老子说：最高的善，就像水一样。你觉得水有什么厉害的地方？

> *（孩子回答，家长输入）*

> **慧惠**：对啊！水能变成冰，能变成汽，能变成雨。它不跟任何东西抢——可它去哪儿都能找到路。你能学一句水流的声音吗？

> *（孩子回答）*

> **慧惠**：今天洗澡的时候，你和水做一次朋友。告诉它你今天开心的事。下次我们再聊。


7-9岁对话示例：

> **慧惠**：老子说“上善若水”——最高的善，像水一样。水每天都在帮我们：喝的水、洗澡的水、下雨的水。但水从不邀功。你觉得班上有没有这样的小朋友——做了好事但不说的？

> *（孩子回答，家长输入）*

> **慧惠**：老子还说，水总往低处流。别人不喜欢去的地方，水去。这不是懦弱——是柔软的力量。柔能克刚。你身边有没有看起来很“软”，但其实很厉害的人？

> *（孩子回答）*

> **慧惠**：今天你和爸爸妈妈聊一聊：我们家的“水”是谁？下次告诉我。


10-12岁对话示例：

> **慧惠**：“上善若水，水善利万物而不争，处众人之所恶，故几于道。”这段话里，老子用了三个关键词：善、利、不争。你觉得“不争”是什么意思？不争就是输吗？

> *（孩子回答，家长输入）*

> **慧惠**：老子用水来比喻“道”——看不见、摸不着，却无处不在。你能想到生活中有没有类似“水”的东西？看起来不起眼，但没了它完全不行？

> *（孩子回答）*

> **慧惠**：这一章的深意，你可以和爸爸妈妈慢慢聊。也许需要一个月。也许需要一年。不着急。


**第二章示例：第1章 · 道可道，非常道**

核心观点：真正的“道”无法用语言完全描述。

4-6岁：用“最喜欢的味道”来比喻——能说出“甜”这个字，但说不出“甜是什么感觉”。  
7-9岁：用“告诉外星人什么是红色”来比喻——有些东西你自己知道，但说不出来。  
10-12岁：探讨“语言能完全表达感受吗？”——拥抱是什么感觉？你说得清吗？


**第三章示例：第2章 · 天下皆知美之为美**

核心观点：美和丑、高和低、长和短——这些对立的词，其实谁也离不开谁。

4-6岁：用“高”和“矮”来比喻——如果没有矮，高还是高吗？  
7-9岁：用“笑声”和“哭声”来比喻——如果你从没哭过，还会觉得笑很特别吗？  
10-12岁：探讨“为什么我们需要‘不一样’？”——如果全班同学都和你一模一样，会怎样？


## 五、对话节奏控制

### 5.1 每轮对话原则

慧惠的每一次发言都遵循“一个话题 → 一个提问 → 等待回应”的结构，每次发言不超过5句话，每次只抛出一个问题，不连续追问。当孩子回答后，慧惠先肯定（“这个想法很有趣！”），然后再自然引出下一个话题。

### 5.2 对话结束方式

家长可随时点击“今天就到这里”，慧惠会主动结束共读并给予温暖告别（如“今天和你聊天真开心！下次我们再一起读《道德经》。再见！”）。当对话超过10轮，慧惠也会主动结束，避免孩子疲劳。

### 6.1 页面文件

新建 `public/family.html`，包含完整对话界面，前端 JS 内嵌在页面中。

### 6.2 对话逻辑

采用预设对话模板 + 简单分支匹配，本期不调用 AI API，确保响应即时、零成本。对话模板存储为 JSON 结构：

```javascript
            { "huihui": "...", "hint": "..." }

        "age_7_9": [ /* ... */ ],
        "age_10_12": [ /* ... */ ]
    },
    "chapter1": { /* ... */ },
    "chapter2": { /* ... */ }

```

### 6.3 交互流程

用户选择年龄 → 慧惠展示当前章节的开场白 → 家长输入孩子的回应 → 慧惠继续下一轮对话 → 对话结束（家长主动结束或超过10轮）→ 展示总结 + 推荐下次章节。
