## 四、核心设计：元数据结构

这是一个极其精简的结构。针对这 3 章试点，你需要花费极少的时间，将核心内容填入此结构中。

### 4.1 元数据 JSON 结构

```json
{
  "chapter": 8,

  "core_idea": "最高的善像水一样，利万物而不争",
  "safety_notes": [

    {

      "age_4_6": null,

      "age_10_12": "老子说'天下之至柔，驰骋天下之至坚'——你能想到什么例子？"
    }
  ],

  "review_status": "approved",
  "reviewed_by": "产品经理",
  "reviewed_at": "2026-05-11"
}
```

### 4.2 字段说明

- `chapter`：章节号（必填）
- `title`：章节标题（必填）
- `core_idea`：核心观点，用于 System Prompt 的上下文注入（必填）
- `safety_notes`：安全提示，会被写入 System Prompt 作为“禁止行为”（必填）
- `interaction_points`：亲子互动点，每个点包含三个年龄段的引导方向。`null` 表示该年龄段不适合此话题（必填，至少1个）
- `parent_tips`：给家长的温暖提示，会在对话结束时展示（必填）
- `review_status`：审核状态。`pending`（待审核）或 `approved`（已通过）。只有 `approved` 的章节才会被 API 使用（必填）
- `reviewed_by`：审核人（选填）
- `reviewed_at`：审核日期（选填）


## 五、API 设计

**仅有 1 个端点**：`POST /api/family_chat`

**请求 Body**：
```json
{
  "chapter": 8,
  "age_group": "age_4_6",
  "conversation_history": [
    {"role": "huihui", "content": "你觉得水有什么厉害的地方？"},
    {"role": "user", "content": "水能变成冰！"}
  ]
}
```

**响应 Body**：
```json
{
  "huihui_response": "对啊！水能变成冰，还能变成汽，变成雨...（动态生成的对话内容）"
}
```

**System Prompt 自动构建逻辑**：

1. 根据 `chapter` 读取元数据，提取 `core_idea`、`safety_notes`、`interaction_points` 中对应 `age_group` 的引导方向
2. 根据 `age_group` 选择对话风格约束（如 4-6岁：用大自然、小动物比喻，禁止复杂词汇）
3. 拼接上亿惠的基础人格定义和亲子陪伴风格
4. 发送至 DeepSeek API，返回动态生成的对话内容


## 六、成本控制与缓存策略

**缓存键格式**：`family_chat_{chapter}_{age_group}_{round}`  
**缓存位置**：Cloudflare Workers KV 或 Vercel Edge Config  
**缓存 TTL**：24小时  
**缓存存储内容**：LLM 生成的每一轮对话文本（含慧惠的发言和给家长的提示）

**成本估算**：当 81 章全量覆盖、3 个年龄段时，首次生成的全部对话缓存量级可控。若按每日 100 个活跃用户、平均每人 5 轮对话计算，其中约 70%-85% 的请求可直接从缓存读取，仅少量新章节或新话题请求调用 LLM，月成本极低。

**成本优化**：
- 闲置超过 30 天的章节对话缓存自动清理，回收存储空间
- 运行 3 个月后，绝大部分高频章节对话已命中缓存，LLM 调用量趋近于零，边际成本极低
