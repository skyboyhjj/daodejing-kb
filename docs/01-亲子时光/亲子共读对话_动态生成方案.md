## 二、双学习模式架构

### 2.1 连续学习模式（`continuous`）

**设计意图**：1→81 章顺序推进，构建持久的学习旅程。强调循序渐进、未完待续的陪伴感。

**核心机制**：

```
用户选择年龄
  ↓
从 localStorage 恢复进度（如有）
  ↓
从第一个未完成章节开始
  ↓
startChapter() → API 动态生成开场白
  ↓
对话进行（最多 10 轮）
  ↓
endChapter() → 标记完成 → 自动跳下一章
  ↓
goToNextChapter() → chapterNum++ → 循环
  ↓
到达 81 章 → allChaptersDone() → 庆祝告别
```

**关键行为**：
- **进度条**：`#chapter-progress` 实时显示 "已完成 X / 81 章 · 当前：第 Y 章"
- **换话题**：`showContinuousChapterSelector()` — 仅显示当前章起的 3 张相邻卡片（.current 高亮）
- **未审核章**：`handleUnapprovedChapter()` — 降级为五步读解法链接卡片 + 自动跳过
- **章节完成**：`endChapter()` 中连续模式自动 `setTimeout → goToNextChapter()`（2 秒过渡）

### 2.2 自由探索模式（`free`）

**设计意图**：任意章节跳转，全部 81 章可见。适合已有基础的用户、想重温特定章节的场景。

**核心机制**：

```
用户选择年龄
  ↓
显示章节号输入框（1-81）
  ↓
用户输入章节号 / 点击"浏览全部 81 章"
  ↓
selectChapterByNum() → startChapter() → API 对话
  ↓
对话结束 → showAllChaptersSelector()
  ↓
13 主题组 × 81 章网格（RTL 横向排列，主题名右侧）
  ↓
选择新章节 → 循环
```

**关键行为**：
- **章节输入**：`#chapter-num-input` — 直接输入 1-81 章节号
- **全量选择器**：`showAllChaptersSelector()` — 13 主题组 RTL 网格，已审核/未审核/已完成 状态区分
- **换话题**：同样显示全量 81 章选择器

### 2.3 模式切换

`switchMode(mode)` 函数根据当前 `state.stage` 自适应处理：

| 当前阶段 | 切换方向 | 处理逻辑 |
|---------|---------|---------|
| `welcome` | 任意 | 更新模式 UI 指示 |
| `intro`（已选年龄） | free→continuous | 隐藏输入区 → 加载进度 → `startChapter()` |
| `intro`（已选年龄） | continuous→free | 恢复 `welcomeEl` 可见 → 显示章节输入 |
| `dialogue`（对话中） | 任意 | 取消加载 → 保存进度 → 结束对话 → 按新模式过渡 |
| `chapter_end` | →continuous | 隐藏选择器 → `goToNextChapter()` |
| `chapter_end` | →free | 显示全量 81 章选择器 |

## 四、API：`POST /api/family_chat`

### 4.1 接口规格

**文件**：`api/family_chat.js`（Vercel）/ `functions/api/family_chat.js`（Cloudflare Pages）

**请求**：
```json
{
    "chapter": 8,
    "age_group": "age_4_6",
    "conversation_history": [
        { "role": "huihui", "content": "你觉得水有什么厉害的地方？" },
        { "role": "user", "content": "水能变成冰！" }
    ]
}
```

**响应**：
```json
{
    "huihui_response": "对啊！水能变成冰，还能变成汽...",
    "cached": false
}
```

**错误码**：
- `400` — 缺少必填参数
- `403` — 章节元数据未审核（前端降级处理）
- `404` — 未找到章节元数据
- `500` — API 调用失败 / 超时

### 4.2 System Prompt 构建

`buildFamilySystemPrompt(chapterMeta, ageGroup)` 动态拼接 5 个模块：

| 模块 | 来源 | 作用 |
|------|------|------|
| 慧惠人格 | 硬编码 | 温柔陪伴者定位，非老师非测试官 |
| 核心观点 | `chapterMeta.core_idea` | 注入章节内容上下文 |
| 年龄风格 | `AGE_STYLE[ageGroup]` | 控制语言难度、篇幅、互动方式 |
| 安全约束 | `chapterMeta.safety_notes[]` | 禁止触及的话题列表 |
| 互动引导 | `chapterMeta.interaction_points[]` | 各年龄段引导方向和话题建议 |

### 4.3 年龄适配风格

| 年龄段 | 语言特点 | 每次篇幅 | 互动方式 |
|--------|---------|---------|---------|
| `age_4_6` | 大自然、小动物、生活比喻，禁止抽象术语 | 1-2 句 | 手指、学叫、画画 |
| `age_7_9` | 小故事、角色扮演、简单类比 | 2-3 句 | 说出自己的故事 |
| `age_10_12` | 引入原文和抽象概念 | 2-3 句 | 思辨性提问 |

### 4.4 审核安全守卫

API 级别强制校验：
```javascript
if (chapterMeta.review_status !== 'approved') {
    return res.status(403).json({ error: '...' });
}
```

前端接收到 403 时自动触发 `handleUnapprovedChapter()` — 降级为五步读解法链接卡片。

## 五、元数据：`data/family_metadata.json`

### 5.1 结构

```json
{
    "chapters": {
        "8": {
            "chapter": 8,
            "title": "上善若水",
            "core_idea": "最高的善像水一样，利万物而不争",
            "safety_notes": ["避免用洪水比喻", "不涉及溺水话题"],
            "interaction_points": [{
                "topic": "水的厉害之处",
                "age_4_6": "水能变成什么？学一个水的声音",
                "age_7_9": "你身边有没有像水一样的人？",
                "age_10_12": "你觉得'不争'是什么意思？"
            }],
            "parent_tips": "和孩子聊的时候，不用刻意讲道理...",
            "review_status": "approved",
            "reviewed_by": "产品经理",
            "reviewed_at": "2026-05-11",
            "review_history": [...]
        }
    }
}
```

### 5.2 审核状态机

```
pending → reviewing → approved
                    → revision_needed → reviewing → ...
```

**状态含义**：
- `pending`：AI 生成，未审核
- `reviewing`：审核进行中（软锁定，防止多人同时编辑）
- `approved`：已通过，API 可用
- `revision_needed`：需修改，含修改建议

## 六、缓存架构

### 6.1 运行时缓存（api/family_chat.js 内置）

- **存储**：`Map<String, {data, ts}>`（内存）
- **键格式**：`fc_{chapter}_{ageGroup}_{historyHash}`
- **TTL**：24 小时
- **命中率**：同一章、同一年龄、相同对话历史 → 直接返回缓存

### 6.2 预热脚本（scripts/warmup_cache.py）

**工作流程**：

```
读取 family_metadata.json → 筛选已审核章节
  ↓
启动本地 server.js（自动查找可用端口）
  ↓
遍历 章节 × 3年龄段 × N轮次
  ↓
调用 http://127.0.0.1:{port}/api/family_chat
  ↓
模拟孩子回答追加到对话历史
  ↓
保存到 data/family_chat_cache.json
```

**命令行用法**：
```bash
python scripts/warmup_cache.py                    # 预热全部已审核章节
python scripts/warmup_cache.py --rounds 2         # 只预热前 2 轮
python scripts/warmup_cache.py --chapters 3,7,9   # 指定章节
python scripts/warmup_cache.py --no-server        # 使用已有服务器
```

### 6.3 元数据缓存（前端 family.js）

```javascript
function loadMetadataCache(callback) {
    if (state.metadataCache) { callback(); return; }
    // GET /data/family_metadata.json
    // 缓存到 state.metadataCache，跨页面生命周期复用
}
```
