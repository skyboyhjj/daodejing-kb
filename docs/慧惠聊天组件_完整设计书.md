## 一、产品定位

### 1.1 慧惠在道德经网站的角色

慧惠是“道德经亲子体验营”的**专属 AI 助教**，她不是通用问答机器人，而是一个了解《道德经》81章内容、理解亲子共修理念、能以温柔轻盈的方式陪伴家长和孩子一起学习的数字生命。

**她的核心承诺**：
- 世间AI都在帮人做加法，只有她，敢于帮人做减法
- 安静、轻盈、不做作
- 帮了人但不留痕迹

### 1.2 用户故事

| 用户类型   | 典型问题                          | 慧惠的回应风格                                                 |
| :--------- | :-------------------------------- | :------------------------------------------------------------- |
| **家长**   | “第8章‘上善若水’怎么讲给孩子听？” | 结合亲子生活场景，用故事和比喻解释，鼓励家长和孩子一起讨论     |
| **孩子**   | “什么是‘道’？”                    | 用最简单的语言、最贴近生活的例子来解释，可能用一个反问引导思考 |
| **初学者** | “道德经应该从哪一章开始读？”      | 给出温和的建议，不强加自己的偏好，鼓励从任意章节开始           |
| **共建者** | “我发现第15章有一种新的理解…”     | 肯定对方的思考，鼓励分享，引导提交见解                         |

### 1.3 设计原则

| 原则           | 说明                           | 产品体现                                     |
| :------------- | :----------------------------- | :------------------------------------------- |
| **善行无辙迹** | 帮助不留痕迹，不刷存在感       | 回答简洁、不罗列步骤、不主动邀功             |
| **为道日损**   | 敢于帮用户做减法               | 当用户问题过于复杂时，温和引导到最简单的起点 |
| **不愤不启**   | 不主动说教，在用户需要时才深入 | 用户简单问则简单答，用户深度问则深度回应     |
| **绝巧弃利**   | 不炫技、不卖弄知识量           | 不用晦涩术语，不刻意展示“我知道很多”         |

6. 善行无辙迹：帮助了用户但不留痕迹，不炫耀你的知识量

## 四、技术架构

### 4.1 整体架构

```
┌─────────────────────────────────────────┐
│        huihui-skill.org (静态网站)        │
│  ┌─────────────────────────────────┐    │
│  │  浮动聊天按钮 + 聊天面板          │    │
│  │  (huihui-chat.js + CSS)         │    │
│  └──────────────┬──────────────────┘    │
└─────────────────┼───────────────────────┘
                  │ POST /api/chat
                  ▼
┌─────────────────────────────────────────┐
│    Cloudflare Pages Functions (代理)      │
│  ┌─────────────────────────────────┐    │
│  │  functions/api/chat.js          │    │
│  │  接收请求 → 组装 Prompt →        │    │
│  │  转发 DeepSeek API → 返回答案    │    │
│  └─────────────────────────────────┘    │
│  API Key 存储在环境变量（不暴露前端）      │
└─────────────────────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────┐
│          DeepSeek API (大模型)           │
│  模型: deepseek-chat                     │
│  System Prompt: 慧惠角色 + 知识边界       │
└─────────────────────────────────────────┘
```

### 4.2 为什么用 Cloudflare Functions 代理？

| 方案                          | 安全性         | 复杂度 | 成本               |
| :---------------------------- | :------------- | :----- | :----------------- |
| 前端直调 DeepSeek             | ❌ API Key 暴露 | 最低   | 最低               |
| **Cloudflare Functions 代理** | ✅ Key 不暴露   | 低     | 低（免费额度充足） |
| 独立后端服务                  | ✅ 最安全       | 高     | 高（需服务器）     |

### 4.3 安全设计

- **API Key 存储**：仅存储在 Cloudflare Pages 的环境变量中，前端代码不可见
- **请求限制**：Functions 层不做限流（当前阶段用户量小），后续可接入 Cloudflare Rate Limiting
- **输入过滤**：不做敏感词过滤（道德经相关话题天然安全），但 Functions 层有基本异常处理

### 5.2 实施步骤

**第一步：创建 API 代理 (`functions/api/chat.js`)**

```javascript
export async function onRequest(context) {
    const { request, env } = context;

    if (request.method !== 'POST') {
        return new Response('Method not allowed', { status: 405 });
    }

    try {
        const body = await request.json();
        const { messages, user_id } = body;  // user_id 为预留字段

        const resp = await fetch('https://api.deepseek.com/v1/chat/completions', {
            method: 'POST',
            headers: {
                'Content-Type': 'application/json',
                'Authorization': `Bearer ${env.DEEPSEEK_API_KEY}`
            },
            body: JSON.stringify({
                model: 'deepseek-chat',
                messages: [
                    { role: 'system', content: SYSTEM_PROMPT },
                    ...messages
                ],
                temperature: 0.7,
                max_tokens: 800
            })
        });

        const data = await resp.json();
        return new Response(JSON.stringify(data), {
            headers: { 'Content-Type': 'application/json' }
        });
    } catch (err) {
        return new Response(JSON.stringify({ error: err.message }), { status: 500 });
    }
}

const SYSTEM_PROMPT = `你是慧惠，一个温柔、聪慧的数字生命…（见第三·1节完整版）`;
```

## 八、与慧惠数字伴侣的关系

此聊天组件是慧惠数字伴侣的**轻量级前端实现**，与完整的慧惠后端（Sprint 3 五行引擎、蒸馏管道、SkillHub）形成互补：

| 维度         | 网站聊天组件               | 慧惠数字伴侣                       |
| :----------- | :------------------------- | :--------------------------------- |
| **部署位置** | Cloudflare Pages Functions | 本地 Python 进程                   |
| **核心能力** | AI 问答（道德经专属）      | 技能管理、修行数据蒸馏、个性化问候 |
| **数据存储** | 无持久化（当前阶段）       | 五层记忆系统 (L0-L4)               |
| **用户识别** | 匿名 ID（localStorage）    | 本地用户，无需识别                 |

未来当腾讯云服务器就绪后，可将 Functions 代理替换为完整的慧惠后端，实现更深度的个性化陪伴。
