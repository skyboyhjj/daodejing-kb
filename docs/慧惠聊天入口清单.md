# 慧惠聊天功能 · 全站入口清单

> 生成日期：2026-05-27

---

## 概览

全站共 **7 个入口**（4 个 HTML onclick 入口 + 1 个 JS 事件监听 + 1 个 SkillUP 程序调用 + 1 个全局函数），分布在 3 个 HTML 页面和 2 个 JS 文件中。

## 二、调用链路图

```
入口 1,4,6 (openHuihuiChat) ─┐
入口 3   (openChatWithPreset)─┤
                              ├──→ btn.click() ──→ chatBtn 'click' listener ──→ openPanel()
                              │                                    (huihui-chat.js:192)
入口 7   (openHuihuiFeedback)─┘
                              (直接在 huihui-chat.js 中调用 openPanel)

入口 5 (浮动按钮) ──→ chatBtn 'click' listener ──→ openPanel()
```

关键机制：入口 1-4、6 都通过**程序化点击** `#huihui-chat-btn` 浮动按钮来触发面板。入口 5 是用户直接点击该按钮。入口 7 在 `huihui-chat.js` 内部直接调用 `openPanel()`。

## 三、各入口实现差异

### 入口 1：首页 "问慧惠" 卡片 (`index.html:373`)

- 四层防御：panelJustOpened 防竞态 → DOM 重试 → 面板状态验证 → 手动补救
- 无预设消息，仅打开面板
- 代码位置：`index.html:429-482`

### 入口 2：首页 "反馈与建议" 链接 (`index.html:414`)

- 直接调用 `openPanel()`（不经过 `btn.click()`），无重试机制
- 显示三类反馈按钮：报错 / 建议 / 帮助
- 用户选择后通过 `[FEEDBACK:SOP=bug/suggestion/help]` 前缀注入 System Prompt
- 代码位置：`huihui-chat.js:807-844`

### 入口 3：L1 儿童版 "问慧惠" 卡片 (`l1/index.html:402`)

- 预设消息后置模式：先确保面板打开 → 再填充输入框并自动发送
- `openHuihuiChat()` 的两个成功路径（btn 命中 + 重试成功）都调用 `maybeSendPreset()`
- Event 构造函数兼容微信浏览器（createEvent 降级）
- 代码位置：`l1/index.html:454-537`

### 入口 4：章节总览页 "打开聊天框" (`chapters.html:819`)

- 最简实现：无重试、无验证、无容错
- 假设 `huihui-chat.js` 已完成 DOM 注入
- 代码位置：`chapters.html:834-837`

### 入口 5：浮动按钮 (`huihui-chat.js:192`)

- 唯一由用户直接点击的原生入口
- 所有程序化入口最终通过 `btn.click()` 触发此监听器
- 设有 `panelJustOpened`（300ms 窗口）防止冒泡事件误关闭面板

### 入口 6：SkillUP "我的成长" 卡片 (`js/skillup-ui.js:107`)

- 仅在未授权状态的卡片上生效
- `__skillupCardTrigger` 标记让 `openPanel()` 在欢迎消息后立即发送许可征求消息（绕过 3 天等待期）
- 授权后自动刷新卡片为成长数据视图

### 入口 7：全局反馈函数 (`window.openHuihuiFeedback`)

- 挂在 window 上的全局函数，供 index.html 页脚链接调用
- 与入口 5 不同：此函数在打开前先设置反馈 UI，消除工具栏闪现问题

## 五、覆盖范围

- 81 个章节页面 (`chapters/ch01.html` ~ `ch81.html`) 和概念页面 (`concepts/`)：均引入 `huihui-chat.js` + CSS，浮动按钮（入口5）在所有页面可用，但没有页面级 onclick 入口
