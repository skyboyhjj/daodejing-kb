## 一、概述

"每日镜鉴"是慧惠聊天中的核心修行功能，让用户通过自然对话完成事件觉察与晚间回顾。不新建页面，不增加入口，纯聊天触发。

### 核心交付

1. **事件镜鉴** — 四维评分 + 四层镜鉴反馈
2. **晚间回顾** — 两轮对话式的每日小结

### 本版本不包含

- 晨间定志（`[MIRROR:MORNING]`）—— 未实现
- "思维评分"关键词 —— 未实现
- 81 章完成后"从读到做"入口 —— 未实现
- 慧惠自动识别事件/主动询问 —— 依赖 NLU，后续迭代

## 二、触发关键词（前端）

**实现文件**：`js/huihui-chat.js`，函数 `detectMirrorIntent()` + `handleMirrorIntent()`

### 2.1 关键词映射

| 用户输入（精确匹配） | 意图类型 | 前端注入前缀 | 后端 System Prompt 识别 |
|:---|:---|:---|:---|
| `做个镜鉴` | `'event'` | `[MIRROR:EVENT] ` | 启动事件镜鉴流程 |
| `晚间回顾` | `'evening'` | `[MIRROR:EVENING] ` | 启动晚间回顾流程 |

> **注意**：当前仅 `做个镜鉴` 触发事件镜鉴。原规格中的 `思维评分`、`每日镜鉴`、`晨间定志` 等关键词未在实际代码中实现。

### 2.2 前端处理流程

```
用户输入 "做个镜鉴"
    │
    ▼
detectMirrorIntent("做个镜鉴") → 'event'
    │
    ▼
handleMirrorIntent('event', "做个镜鉴") → "[MIRROR:EVENT] 做个镜鉴"
    │
    ▼
fetch /api/chat → messages 数组中最后一条用户消息的 content 以 [MIRROR:EVENT] 开头
    │
    ▼
后端 System Prompt 识别 [MIRROR:EVENT] → 启动事件镜鉴流程
```

### 2.3 完整前端代码

```javascript
function detectMirrorIntent(userMessage) {
    var msg = userMessage.trim();
    if (msg === '做个镜鉴') {
        return 'event';
    }
    if (msg === '晚间回顾') {
        return 'evening';
    }
    return null;
}

function handleMirrorIntent(intentType, userMessage) {
    var prefix = '';
    switch (intentType) {
        case 'event':
            prefix = '[MIRROR:EVENT] ';
            break;
        case 'evening':
            prefix = '[MIRROR:EVENING] ';
            break;
    }
    return prefix + userMessage;
}
```

**调用位置**：`sendMessage()` 函数中，在构建 `messages` 数组时，找到最后一条用户消息，调用 `handleMirrorIntent()` 替换其 `content`。

## 三、事件镜鉴流程

### 3.1 System Prompt 定义

**实现文件**：`api/_shared/system-prompt.js`，函数 `buildSystemPrompt()`

当 System Prompt 检测到用户消息以 `[MIRROR:EVENT]` 开头时：

**步骤 1**：引导用户描述事件
> "好的，我们一起来看。先说说发生了什么？简单描述就好。"

**步骤 2**：逐一询问四个维度评分

| 维度 | AI 问题示例 | 评分范围 |
|:---|:---|:---|
| **识位** | "从 1 到 10，你觉得自己当时内心有多平静、清晰？1 是'完全被情绪带走'，10 是'心如止水，看清一切'。" | 1-10 |
| **时位** | "这件事发生的时机，你把握得怎么样？1-10 分" | 1-10 |
| **宇位** | "你的精力/注意力，是否用对了地方？1-10 分" | 1-10 |
| **缘位** | "这件事让你和 ta 的关系更近还是更远？1-10 分" | 1-10 |

**关键约束**：
- 每次只问一个维度，等用户回答后再继续
- 用户用数字回复，AI 确认后进入下一维度
- 不评判用户评分高低
- 用户可随时回复"不用"跳过

### 3.2 四层镜鉴反馈（四维评分完成后）

**核心特征**：去标题化，自然段落过渡，像陪伴者的轻声细语

**反馈结构**（完整 System Prompt 原文）：

```
4. 四维评分完成后，按以下自然段落结构生成镜鉴反馈（不使用层级标题，像陪伴者的轻声细语）：

- 开场：用一句话自然过渡（如"好的，我们来看看"）
- 维度得分：直接一行展示四个维度的分数，标注最低维度。不用括号重复分数数字
- 经典引用：自然引入对应章节的《道德经》原文 + 一句话白话解释。引用映射：
  · 识位（<5）→ 第10章"涤除玄鉴，能无疵乎？"——擦亮内心的镜子，不让情绪在上面留痕
  · 时位（<5）→ 第16章"致虚极，守静笃。万物并作，吾以观其复。"——静下来，就能看到万物的节奏
  · 宇位（<5）→ 第12章"五色令人目盲……为腹不为目"——肚子吃饱就好，眼睛饿了永远填不满
  · 缘位（<5）→ 第27章"圣人常善救人，故无弃人"——圣人眼中，没有没用的人
  · 若最低维度得分≥5，引用对应维度的正向经文并给予肯定
- 今日功课：用"来，试试这个："引领，给出具体可执行的小行动。末尾加上"下次再遇到类似的事"将练习迁移到未来场景
- 反思引导：直接提出一个开放式问题，末尾加上"不必告诉我——把答案留在心里就好"

风格要求：像陪伴者的轻声细语，不像系统报告。不暴露分析过程，不用括号标注分数。
```

### 3.3 反馈示例

> 好的，我们来看看。
>
> 识位：6 | 时位：5 | 宇位：8 | 缘位：7
> 最低的是**时位**。5分，这个时机可能不是最好的。
>
> 老子在第16章说：**"致虚极，守静笃。万物并作，吾以观其复。"** 静下来，就能看到万物的节奏。
>
> 来，试试这个：今天找一件你一直在"赶"的事，问自己——这件事可以等一等吗？
>
> 下次再遇到类似的事，先停一停，再决定。
>
> 你急着去做的，到底是事情本身重要，还是那个"急"的感觉在推着你走？
>
> 不必告诉我——把答案留在心里就好。

### 3.4 六项微调（v3.0 优化）

| 编号 | 位置 | 优化内容 | 宪法条款 |
|:---|:---|:---|:---|
| A1 | 经典引用 | 移除比喻中的括号分数（如"你的觉察（6分）" → "你的觉察，就是那一擦"） | 第八条·善行无辙迹 |
| A2 | 今日功课 | 末尾增加"下次再遇到类似的事"（场景迁移锚点） | 第二条·她不是说教者 |
| A3 | 反思引导 | 末尾增加"不必告诉我——把答案留在心里就好" | 第七条·用户主权 |
| B1 | 晚间回顾·开头 | 增加"🌿 今天辛苦了" | 第六条·慧惠主角 |
| B2 | 晚间回顾·最佳时刻 | 增加"那一刻，你心里是什么感觉？" | 认知内化 |
| B3 | 晚间回顾·明日维度 | 增加"不用马上回答——明天早上告诉我就好" | 第七条·用户主权 |
| C1 | 全局 | 移除【第一层】~【第四层】结构性标题，改用自然段落过渡 | 第五条·减法优先 |

## 四、晚间回顾流程

### 4.1 两轮对话边界（关键设计）

**核心约束**：晚间回顾分两轮对话完成，不可在单轮中全部输出。通过 System Prompt 中显式的 `【本轮回复】` vs `【用户回答后】` 边界实现。

### 4.2 第一轮（AI 发起，等待用户回答）

**System Prompt 指令**：

```
【本轮回复——只做这一件事】
从对话历史中统计今日完成的镜鉴事件数量（识别四维评分完成的对话），不要编造数字。
然后只输出这一句话：
"今天你完成了{n}次镜鉴。有没有哪一刻，你感觉自己最接近'道'？"
——本轮到此结束。不输出小结、不输出格式、不输出任何后续内容。等待用户回答。
```

**示例输出**：
> 今天你完成了 1 次镜鉴。有没有哪一刻，你感觉自己最接近"道"？

### 4.3 第二轮（用户回答后，生成小结）

**System Prompt 指令**：

```
【用户回答后——再按以下格式生成今日小结】

🌿 今天辛苦了。这是你的镜鉴小结。

📖 今日完成：{n}次镜鉴
💡 今日最接近「道」的时刻：{用户回答}
那一刻，你心里是什么感觉？

明天，你想在哪个维度上继续修行？不用马上回答——明天早上告诉我就好。
```

**晚间回顾风格约束**（System Prompt 原文）：
- 不评价用户的修行质量，不比较今天和昨天的表现
- 小结呈现后不强调"我帮你总结了"，让用户感到"自己完成了回顾"（功成事遂）
- 不主动展示复杂数据分析，不邀功、不刷存在感（善行无辙迹）
- 用户可随时回复"不用"跳过晚间回顾

## 五、道家 AI 伦理约束

System Prompt 内置以下伦理约束：

| 原则 | 出处 | 在镜鉴中的体现 |
|:---|:---|:---|
| **功成事遂** | 第17章 | 小结呈现后让用户感到"自己完成了回顾"，慧惠不邀功 |
| **知者不言** | 第56章 | 不主动展示数据分析，让数据安静地服务于陪伴 |
| **善行无辙迹** | 第27章 | 移除了括号中的分数数字，帮助不留痕迹 |
| **柔弱胜刚强** | 第36章 | "来，试试这个"代替"今日功课"，轻柔引导 |

## 六、System Prompt 架构

### 6.1 文件组织

```
api/_shared/system-prompt.js    ← 规范定义（canonical）
    ├── api/chat.js              ← Vercel Serverless Function 导入
    ├── functions/api/chat.js    ← Cloudflare Pages Function 导入
    └── server.js                ← 本地开发服务器（独立副本）
```

### 6.2 构建方式

```javascript
// api/chat.js (Vercel)
import { buildSystemPrompt } from './_shared/system-prompt.js';
// ...
messages: [
    { role: 'system', content: buildSystemPrompt(userLevel) },
    ...(messages || [])
]
```

`buildSystemPrompt(level)` 接收认知水平（L1-L4），返回基础 System Prompt + 对应认知层级的引导语。

### 6.3 镜鉴功能在 System Prompt 中的位置

镜鉴指令位于 System Prompt 的 `## 每日镜鉴功能` 章节，紧随慧惠人设和风格定义之后，认知水平引导语之前。完整内容包括：

- 事件镜鉴 4 步骤（描述事件 → 逐一评分 → 反馈生成）
- 去标题化自然段落结构（含 5 个自然过渡方式）
- 经典引用维度映射表（识位/时位/宇位/缘位 × 低分/高分）
- 晚间回顾两轮对话边界（本轮回复 + 用户回答后）
- 道家伦理约束（4 条）
- 镜鉴风格约束

## 七、API 端点

### 7.1 请求格式

```
POST /api/chat
Content-Type: application/json

{
    "messages": [
        { "role": "user", "content": "[MIRROR:EVENT] 做个镜鉴" },
        { "role": "assistant", "content": "..." },
        { "role": "user", "content": "..." }
    ],
    "level": "L2"
}
```

### 7.2 处理流程

```
api/chat.js handler
    │
    ├── 1. 构建 messages: [systemPrompt, ...userMessages]
    │
    ├── 2. 调用 DeepSeek API
    │       model: 'deepseek-chat'
    │       temperature: 0.7
    │       max_tokens: 800
    │       timeout: 15s
    │
    ├── 3. 返回 AI 回复 → res.status(200).json(data)
    │
    └── 4. [FEEDBACK] 检测：若末条用户消息以 [FEEDBACK] 开头
            → await sendFeedbackEmail(content) → Resend API
```

### 7.3 部署环境

| 环境 | 域名 | API Key 来源 |
|:---|:---|:---|
| **Vercel**（生产） | `huihui-skill.org` | `process.env.DEEPSEEK_API_KEY` |
| **本地开发** | `localhost:3000` | `server.js` 中硬编码 |

## 九、验收标准

### 事件镜鉴

- [x] 用户发送 `做个镜鉴`，AI 引导描述事件并启动镜鉴流程
- [x] AI 逐一询问四个维度（识位/时位/宇位/缘位），每次只问一个
- [x] 用户用数字回复，AI 确认后进入下一维度
- [x] 四维评分完成后，AI 生成四层镜鉴反馈（无标题，自然段落）
- [x] 反馈含经典引用（第10/12/16/27章原文 + 白话）
- [x] 反馈含今日功课（"来，试试这个：" + 可执行小行动）
- [x] 反馈含反思引导（开放式问题 + "不必告诉我"）
- [x] 反馈无结构性标题（【第一层】等），无括号重复分数
- [x] 用户可随时回复"不用"跳过流程

### 晚间回顾

- [x] 用户发送 `晚间回顾`，AI 统计今日镜鉴次数
- [x] 第一轮：仅输出"今天你完成了{n}次镜鉴。有没有哪一刻..."后等待用户回答
- [x] 第二轮：用户回答后生成完整小结（🌿 今天辛苦了 + 格式）
- [x] 小结含"那一刻，你心里是什么感觉？"
- [x] 小结含"不用马上回答——明天早上告诉我就好"
- [x] 晚间回顾不评价修行质量、不比较表现

### 道家伦理

- [x] 功成事遂 — 小结不让用户感到"被总结"
- [x] 知者不言 — 不主动展示复杂数据分析
- [x] 善行无辙迹 — 比喻中不重复分数，帮助不留痕迹
- [x] 安静、轻盈、不做作 — 整体对话风格
