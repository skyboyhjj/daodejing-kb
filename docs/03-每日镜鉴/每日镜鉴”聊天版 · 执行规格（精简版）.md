## 一、任务概述

在慧惠聊天窗口中实现“每日镜鉴”的核心交互——晨间定志、事件镜鉴、晚间回顾。不新建页面，不增加入口，用户通过聊天自然触发。

### 核心交付

1. 晨间定志引导（用户主动发送关键词触发）
2. 事件镜鉴四维评分与反馈（用户描述事件后触发）
3. 晚间回顾小结生成（用户主动发送关键词触发）

### 触发方式（本阶段仅两种）

| 触发方式 | 具体行为 | 适用场景 |
|:---|:---|:---|
| **用户主动发送关键词** | 在聊天窗口中发送“**做个镜鉴**”或“**思维评分**”，慧惠引导用户描述事件，启动事件镜鉴流程 | 任何有修行意愿的用户 |
| **用户主动发送“晚间回顾”** | 在聊天窗口中发送“**晚间回顾**”，慧惠总结今日事件并生成小结 | 用户想要回顾当天的修行 |


## 二、前端关键词识别逻辑

**操作文件**：`daodejing-kb/js/huihui-chat.js`

在 `sendMessage()` 函数中增加关键词识别：

```javascript
function detectMirrorIntent(userMessage) {
    var msg = userMessage.trim();
    
    // 事件镜鉴触发（精确匹配两个关键词）
    if (msg === '做个镜鉴' || msg === '思维评分') {
        return 'event';
    }
    // 晚间回顾触发
    if (msg === '晚间回顾') {
        return 'evening';
    }
    return null;
}

function handleMirrorIntent(intentType, userMessage) {
    var prefix = '';
    switch (intentType) {
        case 'event':
            prefix = '[MIRROR:EVENT] ';
            break;
        case 'evening':
            prefix = '[MIRROR:EVENING] ';
            break;
    }
    return prefix + userMessage;
}
```

**使用方式**：在发送消息前，调用 `detectMirrorIntent()` 检测是否为镜鉴关键词，若是则自动添加对应的引导标记（`[MIRROR:EVENT]` 或 `[MIRROR:EVENING]`），后端 System Prompt 根据标记启动对应流程。


## 三、交互流程

### 3.1 事件镜鉴（核心功能，2-3分钟）

**触发条件**：用户发送“做个镜鉴”或“思维评分”

**慧惠引导描述**：
> “好的。先告诉我，发生了什么？”

用户描述事件后，慧惠逐一询问四个维度，每次只问一个：

| 维度 | 慧惠的问题 | 评分范围 |
|:---|:---|:---|
| **时位** | “这件事发生的时机，你把握得怎么样？1-10分” | 1-10 |
| **宇位** | “你的精力/注意力，是否用对了地方？1-10分” | 1-10 |
| **识位** | “当时你的内心是清晰的还是被情绪遮蔽的？1-10分” | 1-10 |
| **缘位** | “这件事让你和ta的关系更近还是更远？1-10分” | 1-10 |

**评分方式**：用户用数字回复，慧惠确认后进入下一维度。

**镜鉴反馈**：四维评分完成后，慧惠生成镜鉴反馈：
> “🌿 镜鉴
>
> **时位**：7分。你把握住了时机，但或许可以再等一等。
> **宇位**：5分。这件事消耗了你太多精力。
> **识位**：4分。当时你的内心被‘不耐烦’遮蔽了。
> **缘位**：6分。ta或许感觉到了你的情绪，但你没说破。
>
> 今天的关键词是‘识位’。下一次，当你感到不耐烦时，试着先深呼吸三次。”

### 3.2 晚间回顾（1-2分钟）

**触发条件**：用户发送“晚间回顾”

**慧惠引导**：
> “今天你记录了{事件数}件事。有没有哪一刻，你感觉自己最接近‘道’？”

用户回答后，慧惠生成今日小结：
> “🌿 今日小结
>
> 📖 今日事件：{事件数}件
> 💡 今日最接近‘道’的时刻：{用户回答}
>
> 明天，你想在哪个维度上继续修行？”

```
## 每日镜鉴功能

当用户消息以 [MIRROR:EVENT] 开头时，启动事件镜鉴流程：

### 镜鉴风格
- 安静、轻盈、不做作

- 评分后不比较、不排名
- 每次只问一个问题，等用户回答后再继续
```
