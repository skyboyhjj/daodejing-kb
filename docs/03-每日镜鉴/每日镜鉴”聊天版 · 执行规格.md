## 一、任务概述

在慧惠聊天窗口中实现“每日镜鉴”的核心交互——晨间定志、事件镜鉴、晚间回顾。不新建页面，不增加入口，用户通过聊天自然触发。

### 核心交付

1. 晨间定志引导（用户主动发送关键词触发）
2. 事件镜鉴四维评分与反馈（用户描述事件时触发）
3. 晚间回顾小结生成（用户主动发送关键词触发）
4. 完成81章后的“从读到做”入口联动

### 触发方式（本阶段仅两种）

| 触发方式 | 具体行为 | 适用场景 |
|:---|:---|:---|
| **完成81章后点击“从读到做”** | 在回顾卡片中点击该按钮，自动打开慧惠聊天窗口，并发送预设消息“每日镜鉴”，触发晨间定志流程 | 完成81章用户的自然引导 |
| **用户主动发送关键词** | 在聊天窗口中发送“**每日镜鉴**”或“**晨间定志**”两个关键词之一，慧惠启动对应流程 | 任何有修行意愿的用户 |

**本阶段暂不实现**：慧惠自动识别事件并引导镜鉴、慧惠在傍晚或感知到用户情绪时主动询问。这些功能依赖复杂的NLU逻辑，后续迭代再加入。


## 二、前端关键词识别逻辑

在 `huihui-chat.js` 的 `sendMessage()` 函数中增加关键词识别：

```javascript
function detectMirrorIntent(userMessage) {
    var msg = userMessage.trim();
    
    // 晨间定志触发（精确匹配两个关键词）
    if (msg === '每日镜鉴' || msg === '晨间定志') {
        return 'morning';
    }
    // 晚间回顾触发
    if (msg === '晚间回顾') {
        return 'evening';
    }
    return null;
}

function handleMirrorIntent(intentType, userMessage) {
    var prefix = '';
    switch (intentType) {
        case 'morning':
            prefix = '[MIRROR:MORNING] ';
            break;
        case 'evening':
            prefix = '[MIRROR:EVENING] ';
            break;
    }
    return prefix + userMessage;
}
```

**使用方式**：在发送消息前，调用 `detectMirrorIntent()` 检测是否为镜鉴关键词，若是则自动添加对应的引导标记（`[MIRROR:MORNING]` 或 `[MIRROR:EVENING]`），后端 System Prompt 根据标记启动对应流程。


## 三、交互流程

### 3.1 晨间定志（1-2分钟）

**触发条件**：用户发送“每日镜鉴”或“晨间定志”

**慧惠引导**：
> “早上好。今天你想在哪个维度上特别留意？”

展示四个维度选项：

| 维度 | 关键词 | 含义 |
|:---|:---|:---|
| **时位** | 时机、节奏 | 做事是否合乎时机？ |
| **宇位** | 精力、资源 | 精力是否用对了地方？ |
| **识位** | 清晰、觉知 | 对自己诚实吗？有没有骗自己？ |
| **缘位** | 关系、因果 | 和身边人的关系是建设还是消耗？ |

用户选择后，慧惠生成今日心法：
> “今日心法：知常曰明。做事时，留意时机。”

**特殊处理**：若用户未明确选择，慧惠默认使用“识位”，心法为：“涤除玄鉴。今天，试着对自己诚实一次。”

### 3.2 事件镜鉴（核心功能，2-3分钟）

**触发条件**：在晨间定志完成后，慧惠引导用户描述事件。用户也可以直接发送“每日镜鉴”后描述事件。

**慧惠引导描述**：
> “好的，我们来做一个镜鉴。你刚才提到的事，在四个维度上，你给自己打几分？”

慧惠逐一询问四个维度，每次只问一个：

| 维度 | 慧惠的问题 | 评分范围 |
|:---|:---|:---|
| **时位** | “这件事发生的时机，你把握得怎么样？1-10分” | 1-10 |
| **宇位** | “你的精力/注意力，是否用对了地方？1-10分” | 1-10 |
| **识位** | “当时你的内心是清晰的还是被情绪遮蔽的？1-10分” | 1-10 |
| **缘位** | “这件事让你和ta的关系更近还是更远？1-10分” | 1-10 |

**评分方式**：用户用数字回复，慧惠确认后进入下一维度。

**镜鉴反馈**：四维评分完成后，慧惠生成镜鉴反馈：
> “🌿 镜鉴
>
> **时位**：7分。你把握住了时机，但或许可以再等一等。
> **宇位**：5分。这件事消耗了你太多精力。
> **识位**：4分。当时你的内心被‘不耐烦’遮蔽了。
> **缘位**：6分。ta或许感觉到了你的情绪，但你没说破。
>
> 今天的关键词是‘识位’。下一次，当你感到不耐烦时，试着先深呼吸三次。”

### 3.3 晚间回顾（1-2分钟）

**触发条件**：用户发送“晚间回顾”

**慧惠引导**：
> “今天你记录了{事件数}件事。有没有哪一刻，你感觉自己最接近‘道’？”

用户回答后，慧惠生成今日小结：
> “🌿 今日小结
>
> 📖 今日事件：{事件数}件
> 🎯 今日心法：知常曰明
> 💡 今日最接近‘道’的时刻：{用户回答}
>
> 明天，你想在哪个维度上继续修行？”

```
## 每日镜鉴功能

当用户消息以 [MIRROR:MORNING] 开头时，启动晨间定志流程：
1. 引导用户从四个维度（时位/宇位/识位/缘位）中选择今日重点

3. 四维评分完成后，生成镜鉴反馈（标注最低维度和简短建议）

### 镜鉴风格
- 安静、轻盈、不做作
- 不评判用户的行为，只提供觉察的视角
- 评分后不比较、不排名
- 每次只问一个问题，等用户回答后再继续
```

## 七、数据存储

| 数据类型 | 存储方式 | 用途 |
|:---|:---|:---|
| 今日修行记录（晨间定志、事件镜鉴、晚间回顾） | 聊天窗口的对话历史 | 当日回顾、慧惠上下文感知 |
| 四维评分历史 | 用户浏览器 localStorage（仅在用户同意追踪后） | SkillUP 阶段判定参考 |
| 今日心法 | 聊天窗口对话历史 | 晚间回顾引用 |

**不持久化存储**：用户的事件描述原文、四维评分的具体数值不在服务器端存储。所有修行数据仅在用户本地浏览器中保留（若用户同意追踪），或在聊天窗口中作为对话历史自然存在。
