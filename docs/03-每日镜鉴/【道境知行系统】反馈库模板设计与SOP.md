# 【道境知行系统】反馈库模板设计与SOP

## 一、反馈库模板设计总览

### 1.1 设计哲学
- **镜鉴非教材**：以经为镜，照见用户自身生活事件
- **知行合一**：每条反馈包含完整的“洞察→原理→行动→反思”闭环
- **四维统摄**：每个反馈条目聚焦一个维度（时位/宇位/识位/缘位）
- **动态演化**：反馈库可模块化、版本化、用户自主下载更新

### 1.2 数据模型核心字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| **Chapter** | 整数 | 道德经章节号 |
| **Chapter Title** | 字符串 | 章节标题 |
| **Dimension** | 枚举 | 时位轴 / 宇位轴 / 识位轴 / 缘位轴 |
| **P Level** | 枚举 | 低 / 高 |
| **Trigger Keywords** | 字符串（逗号分隔） | 触发关键词，用于匹配用户输入 |
| **Insight Title** | 字符串 | 洞察标题，精炼点题 |
| **Insight Description** | 字符串 | 镜鉴描述，直接点出用户状态 |
| **Core Principle** | 字符串 | 核心原理，引用经文或道理解释 |
| **Action Title** | 字符串 | 行动标题，微修行的名称 |
| **Action Description** | 字符串 | 具体可行的行动建议 |
| **Action Category** | 枚举 | 观照练习/静默练习/布施练习/关系练习/简化练习/觉察练习/感恩练习 |
| **Reflection Prompt** | 字符串 | 反思问题，引导用户“证”的环节 |
| **Status** | 枚举 | 待填写 / 已完成 / 需修订 |

### 1.3 四维各维度核心关注点

| 维度 | 低P忠恕表现 | 高P忠恕表现 | 核心经文指引 |
|------|-------------|-------------|-------------|
| **时位轴** | 逆势而动、强求发展、无视周期 | 顺应节律、知进知退、因时制宜 | “反者道之动”、“功成身退” |
| **宇位轴** | 占有囤积、资源不公、浪费 | 分享流动、均衡调节、知足 | “损有余补不足”、“为腹不为目” |
| **识位轴** | 心随境转、杂念纷飞、被偏见遮蔽 | 心镜澄明、如实观照、涤除玄鉴 | “致虚极守静笃”、“自知者明” |
| **缘位轴** | 自我中心、关系紧张、零和博弈 | 推己及人、广结善缘、共生共赢 | “善利万物而不争”、“为而不争” |

## 二、反馈条目“知-行-证”闭环结构

每条反馈必须包含以下三个环节（对应Excel中的字段组）：

```
┌─────────────────────────────────────────────┐
│                知（Knowing）                 │
├─────────────────────────────────────────────┤
│ • 触发关键词（用于匹配用户输入）              │
│ • 洞察标题（点题）                          │
│ • 洞察描述（“镜鉴：…”开头的状态解析）        │
│ • 核心原理（经文或道理依据）                  │
├─────────────────────────────────────────────┤
│                行（Doing）                  │
├─────────────────────────────────────────────┤
│ • 行动标题（可操作的微修行名称）              │
│ • 行动描述（具体步骤、时间、对象）            │
│ • 行动类别（分类标签）                       │
├─────────────────────────────────────────────┤
│                证（Proving）                │
├─────────────────────────────────────────────┤
│ • 反思提示（引导用户记录体证的问题）           │
└─────────────────────────────────────────────┘
```

### 2.1 撰写准则
- **洞察描述**：必须使用“镜鉴：”开头；语气慈悲而直接，如挚友提醒；字数控制在60字以内。
- **行动描述**：必须可操作（具体动作+时间+对象）；必须微小（5-15分钟可完成）；不得要求用户“学习《道德经》章节”，要围绕用户自身生活事件。
- **反思提示**：开放式问题，引导深度内省；与行动直接相关。

## 三、触发流程（用户事件 → 镜鉴反馈）

整个流程在用户设备本地完成（或用户授权后上传部分数据）。

```
┌─────────────────────────────────────────────────────────┐
│  Step 1: 用户输入事件描述                               │
│  “刚才会议上，我对下属的提案感到很不耐烦”               │
└─────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────┐
│  Step 2: 本地NLP关键词提取                              │
│  提取出：不耐烦、会议、下属、提案                        │
└─────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────┐
│  Step 3: 关键词匹配反馈库                                │
│  遍历已安装的反馈库（如道德经反馈库）                    │
│  查找“Trigger Keywords”中包含这些词或近义词的条目        │
└─────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────┐
│  Step 4: 用户自评四维分数（可选滑动条）                  │
│  用户对时、宇、识、缘四个维度进行1-10分自评              │
│  最低分维度被标记为“核心关注维度”                        │
└─────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────┐
│  Step 5: 匹配策略（优先级）                             │
│  1. 优先匹配“核心关注维度”的匹配条目                     │
│  2. 若无，则匹配“识位轴”或“缘位轴”的匹配条目            │
│  3. 若仍无，返回默认条目（第1章·识位轴·低P忠恕）         │
└─────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────┐
│  Step 6: 返回镜鉴反馈                                   │
│  包含：状态洞察、行动建议、反思提示                      │
│  同时展示关联的经文和行动类别                            │
└─────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────┐
│  Step 7: 用户践行并记录体证                             │
│  用户完成行动后，根据反思提示记录体证文字                │
│  体证包可选择性上传至社区（匿名）                        │
└─────────────────────────────────────────────────────────┘
```

### 3.1 匹配引擎的规则示例（伪代码）

```python
def match_feedback(user_text, user_scores):
    keywords = extract_keywords(user_text)
    candidates = []
    for keyword in keywords:
        for entry in feedback_database:
            if keyword in entry.triggers:
                candidates.append(entry)
    if not candidates:
        return default_entry
    
    # 找出用户自评最低的维度
    lowest_dim = min(user_scores, key=user_scores.get)
    # 优先筛选该维度的候选
    dim_candidates = [e for e in candidates if e.dimension == lowest_dim]
    if dim_candidates:
        # 再优先选择P水平（根据自评分数高低映射）
        # 如果该维度自评分<5 => 低P；>=5 => 高P
        target_p = '低' if user_scores[lowest_dim] < 5 else '高'
        final = next((e for e in dim_candidates if e.p_level == target_p), dim_candidates[0])
        return final
    return candidates[0]
```

## 四、SOP：从章节到完整反馈库的生成流程

### 4.1 准备工作
- 选定《道德经》章节（建议分轮：核心10章 → 扩展12章 → 完成81章）
- 准备Excel模板（包含上述13列）
- 准备每章的原文及核心主旨简述

### 4.2 逐条填充步骤（每条反馈约5-10分钟）

1. **确定本反馈的维度与P水平**（如：第8章·宇位轴·低P忠恕）。
2. **撰写触发关键词**（5-10个）：列举用户描述该状态时可能用的词（具体、生活化），正面、负面、中性均覆盖。
3. **撰写洞察标题**（4-8字）：如“水滞为淤”。
4. **撰写洞察描述**：以“镜鉴：”开头，点出用户当下状态的心理/行为模式，直接指出问题但无评判。
5. **撰写核心原理**：引用该章最相关的一句原文，或化用其核心道理。
6. **撰写行动标题**（6-10字）：如“做一次‘让利’的尝试”。
7. **撰写行动描述**：具体、微小、可操作，聚焦用户自身生活事件（不是学习经典）。
8. **选择行动类别**：从预设的7个类别中选择最贴切的一个。
9. **撰写反思提示**：开放式问题，引导用户践行后内省。
10. **检查整体**：确保语气慈悲而直接、行动不依赖他人、不与经典强绑定。

### 4.3 质量检查清单（每条反馈必须通过）

- [ ] 触发关键词不包含用户可能不会使用的学术术语
- [ ] 洞察描述中“镜鉴”二字开头，且直接点出状态
- [ ] 核心原理引用准确
- [ ] 行动描述可在15分钟内独立完成
- [ ] 行动描述不涉及“学习《道德经》第X章”
- [ ] 反思提示是开放式的，不是“是/否”问题
- [ ] 整条反馈未出现评判性词汇（如“你错了”、“你应该”）

### 4.4 版本管理规范
- 每个反馈库模块包含 `manifest.json`（元数据：模块ID、版本号、作者、简介、依赖等）
- 每条反馈条目标记创建日期和修订日期
- 重大更新需更新版本号，并通知用户可下载

### 4.5 协作与审核流程（适用于团队或社区）
1. 创作者填写Excel模板（草稿）
2. 提交审核人进行质量检查（对照检查清单）
3. 审核通过后，使用自动化脚本转换为 `.zhmirror` 格式
4. 上传至“道境集市”或官方库服务器
5. 用户端根据版本号自动检查更新（用户手动确认下载）

## 五、未来扩展：UGC与动态演化

### 5.1 用户体证包的上传与利用
- 用户践行后记录的“反思答案”构成本地体证包
- 用户可手动选择将体证包（脱敏后）上传至社区智慧池
- 系统根据体证包的“有帮助”评价和P忠恕分数，动态调整反馈库中对应条目的优先级

### 5.2 第三方智者创建专题反馈库
- 提供“镜鉴工坊”低代码工具，引导按模板填写
- 创作者可定价上架“道境集市”
- 平台对内容进行审核（符合“知-行-证”结构和伦理要求）

### 5.3 反馈库的自进化
- 高信用用户的体证包被采纳为优化反馈库的依据
- 社区投票或“有帮助”率高的行动建议可被提升为官方推荐
