## 1. 任务概述

为“慧惠”项目（`daodejing-kb`）构建**镜鉴数据库自进化引擎的 P0 阶段**。  
核心目标：将用户每次“事件镜鉴”与“晚间回顾”的数据，以结构化方式存储在客户端 `localStorage` 中，并通过增量计算形成个人趋势，同时将维度评分接入 `SkillUP` 成长追踪，形成“认知→水德→镜鉴”的闭环。

**P0 阶段边界**：纯客户端、匿名本地存储，不上传服务器，无账号系统。

---

## 2. 技术约束与宪法条款

| 约束项 | 说明 |
|:---|:---|
| **外挂增强** | 不得修改 `huihui-chat.js`、`family.js` 等的现有核心逻辑，只能通过挂载钩子或新增独立模块增强。 |
| **数据隐私** | 所有数据仅存于用户本地浏览器（`localStorage`），不发起任何网络同步请求。 |
| **减法优先** | 新增代码量最小化；镜鉴记录仅在用户完成镜鉴/回顾时触发，不增加额外交互负担。 |
| **性能** | 存储读写为同步操作或极小异步操作，不得阻塞 UI；趋势更新计算量控制在 O(n) 内。 |
| **容错** | 若 `localStorage` 不可用（如隐私模式），静默降级，不影响核心聊天功能。 |

## 4. 详细需求

### 4.1 镜鉴数据库模块 `mirror-db.js`

#### 4.1.1 存储结构

在 `localStorage` 中统一使用 key `huihui_mirror_db`，其 JSON 结构为：

```json
{
  "version": 1,
  "records": [],
  "trends": {
    "识位": { "scores": [6,5,7], "average": 6.0, "trend": "up" },
    "时位": { "scores": [6,6,6], "average": 6.0, "trend": "stable" },
    "宇位": { "scores": [8,8,8], "average": 8.0, "trend": "stable" },
    "缘位": { "scores": [7,8,8], "average": 7.7, "trend": "up" }
  },
  "lastRecordTimestamp": 1716038400,
  "synced_to_cloud": false
}
```

每条 `record` 结构（遵循对话启动包 5.3 节）：

```json
{
  "mirror_id": "mirror_20260518_001",
  "session_id": "session_20260518_chat",
  "timestamp": 1716038400,
  "type": "event_mirror",  // "event_mirror" 或 "evening_review"
  "dimension_scores": { "识位": 6, "时位": 6, "宇位": 8, "缘位": 7 },
  "lowest_dimension": "识位",
  "classic_ref": "第10章·涤除玄鉴",
  "practice": "深呼吸三次，先呼吸再回应",
  "reflection": "放下生气，改为建议",
  "transformation": "从情绪反应到理性建议"
}
```

#### 4.1.2 对外 API

所有方法均挂载在全局对象 `MirrorDB` 下，以便 `huihui-chat.js` 和 `skillup-tracker.js` 调用。

```javascript
// 初始化，检查并升级版本
MirrorDB.init()

// 记录一条镜鉴数据，同时自动调用 updateTrends()
// 参数 recordData 为上面 record 结构（不包含 mirror_id 和 timestamp，由函数自动生成）
MirrorDB.record(recordData) // 返回 mirror_id

// 获取所有记录
MirrorDB.getAllRecords()

// 获取最近 N 条记录（默认 7）
MirrorDB.getRecentRecords(n = 7)

// 获取个人趋势数据
MirrorDB.getTrends()

// 强制更新趋势（通常 record 方法内部会自动调用，但可单独使用）
MirrorDB.updateTrends()

// 导出全部数据为 JSON 字符串（供 P1 迁移使用）
MirrorDB.export()

// 清空所有数据（需用户手动确认，仅提供入口，不主动调用）
MirrorDB.clear()
```

#### 4.1.3 自动 ID 生成规则

`mirror_id` 格式：`mirror_YYYYMMDD_序号`，序号从 `001` 起，按天重置。  
实现方式：扫描当天已有记录数并递增。

#### 4.1.4 增量趋势更新逻辑

`updateTrends()` 遍历所有 `records`，对每个维度：
- 收集所有历史分数数组；
- 计算平均分（保留一位小数）；
- 比较最新两次分数：若最新 > 前一次则 `trend = "up"`，若相等则 `stable`，否则 `down`。

### 4.2 在 `huihui-chat.js` 中挂载记录钩子

**约束**：不修改现有镜鉴生成逻辑，仅在其成功展示结果并得到用户确认后挂入钩子。

**预期触发位置**（由 Qcoder 根据现有代码确定具体行数）：
1. **事件镜鉴完成**：当聊天流中镜鉴内容已展示且用户点击了“留在心里”或类似关闭按钮时。
2. **晚间回顾完成**：当晚间回顾总结已展示，用户结束互动时。

**挂载代码示例**（需适配现有代码）：

```javascript
// 在适当的回调中
if (typeof MirrorDB !== 'undefined') {
  MirrorDB.record({
    session_id: currentSessionId,
    type: 'event_mirror',
    dimension_scores: { 识位: 6, 时位: 6, 宇位: 8, 缘位: 7 },
    lowest_dimension: '识位',
    classic_ref: '第10章·涤除玄鉴',
    practice: '深呼吸三次',
    reflection: userReflection,   // 从交互中提取
    transformation: transformText // 从反馈中提取
  });
}
```

**注意**：记录操作必须放在所有 UI 更新之后，避免阻塞渲染。

### 4.3 SkillUP 联动接口 `skillup-tracker.js`

在 `SkillUPTracker` 对象上新增一个方法，供 `mirror-db.js` 在记录完成后调用（或由 `huihui-chat.js` 一并调用）：

```javascript
SkillUPTracker.linkMirrorDimension(dimensionScores, lowestDimension)
```

**功能**：
- 根据 `lowestDimension` 映射到“自我觉察能力”内部指标。
- 更新成长数据，但不直接改变用户的水德阶段；仅作为辅助参考。
- 若当前水德阶段 >= 3（“上善”），慧惠的镜鉴反馈将更侧重启发式提问；可在 `huihui-chat.js` 中读取该辅助指标来调整反馈风格。

**实现要求**：
- 在 `skillup-tracker.js` 内部维护一个 `mirrorInsightScore` 对象。
- 每次调用时，根据维度分数变化和趋势更新该分数。
- 暴露一个读取方法 `SkillUPTracker.getMirrorInsightLevel()`，返回 `{ score, sufficientForHeuristic: boolean }`。

## 5. 非功能性验收标准

| 测试项 | 预期结果 |
|:---|:---|
| **存储验证** | 完成一次事件镜鉴后，在控制台执行 `MirrorDB.getAllRecords()` 返回刚写入的记录。 |
| **趋势更新** | 连续多次记录后，`MirrorDB.getTrends()` 返回的趋势方向合理。 |
| **ID 生成** | 同一天记录 ID 序号递增，隔天从 001 开始。 |
| **隐私模式** | 若 `localStorage` 不可用，聊天功能不受影响，控制台无报错。 |
| **SkillUP 联动** | `SkillUPTracker.getMirrorInsightLevel()` 返回有效分数。 |
| **代码回归** | 原有聊天功能（事件镜鉴、晚间回顾）正常运行，无功能退化。 |
